## 一、品牌核心表达

### 1.1 一句话定位

**中文**：UniMind.ai 是 Agent 驱动的新一代智能教育基础设施——从教师出题、学生刷题、评分批改到知识追踪，全链路 Agent 化。嵌入，不替代。

**英文**：UniMind.ai is an Agent-driven next-generation education infrastructure — full-chain Agent automation from question generation and grading to knowledge tracking. Embed, don't replace.

### 1.2 电梯演讲（30 秒）

我们不是"给教育产品加了 AI"，而是**用 Agent 重构了教育的底层基础设施**。

UniMind 部署了两个自治 AI Agent：**小宇**是学生唯一的 AI 入口——诊断薄弱点、规划学习路径、讲解知识难点，不等你问就主动分析；**工作台**是教师的教研引擎——一句话出题，背后四个 Agent 对抗博弈，自然语言操作全流程。

底层是 Memorix-Field 自进化记忆调度引擎——融合 Weibull 遗忘模型 + 知识图谱拓扑分析，在线随机梯度下降实时进化每个学生的记忆模型，同时精准定位知识掌握度盲区。

我们有一条硬线：**AI 在执行域，教师在判断域。** 嵌入，不替代。

### 1.3 品牌故事（长版）

传统教育 SaaS 是"功能拼盘"——刷题一个模块、课程一个模块、AI 助手一个聊天框。它们各自为政，学生在割裂的体验中疲于切换。

UniMind 的起点不同。我们从第一天就在构建**基础设施层**：

- **Agent 统一运行时**：所有 AI 能力共享一套 Bot → ToolExecutor → ChatService 架构。新增一个 Agent，只需定义工具集和系统提示词——不需要重建流水线。
- **Memorix-Field 自进化记忆引擎**：融合 Weibull 遗忘模型与知识图谱拓扑分析。每道题的答题行为实时更新个人记忆参数，同时图拓扑定位知识盲区——不只是"哪个知识点弱"，而是"因为哪个前置节点未掌握，导致后续哪些节点理解困难"。记忆衰减状态 + 知识结构依赖，统一决定"下一步该学什么"。
- **对抗式内容生产**：出题不是"AI 生成 → 人工审核"的单次调用。而是 Author、Reviewer、AuthorRevise、Classifier 四个 Agent 迭代博弈，Reviewer 开启深度思考模式，三维质量评分，不达标不放行。
- **MUTAR 自进化闭环**：Measure→Umpire→Think→Adapt→Refine。每次 Agent 对话自动采集轨迹，用户反馈闭环评估，系统自主优化 Prompt 策略。用得越久，Agent 越聪明。
- **嵌入架构，不是替代**：教师设定难度、选题决策、布置时机——这些是判断域，AI 不越界。AI 做搜索、生成、校验、汇总——这些是执行域。两层之间是**硬约束**，不存在绕过教师确认的代码路径。

当我们构建了"小宇"学习教练 Agent 和"工作台"教研 Agent 之后，系统的性质发生了质变——它不再是功能的集合，而是**Agent 的组织**。每个 Agent 有自己的工具链、跨会话记忆、自主决策能力，但共享同一套运行时基础设施。

这就是"新一代智能教育基础设施"的含义：不是给教育加 AI，而是用 AI 重构教育。

## 二、双 Agent 文案

### 2.1 小宇 — 学生学习教练

**一句话**：你唯一的 AI 入口。不等你问，主动分析、规划、讲解。

**完整描述**：
小宇不是问答机器人，而是一个**教练型 Agent**。它能自主调用 21 个工具：查看学习统计、分析知识图谱掌握度、检索到期复习、搜索课程和文章、生成可视化图表，然后综合一切信息给出行动建议。

小宇是学生端的唯一 AI 入口，通过意图路由按需调度工具——不需要在多个 Bot 之间切换，一个 Agent 解决所有问题。

**Memorix-Field 引擎驱动**：融合遗忘建模与知识图谱拓扑分析——基于记忆难度、稳定性、遗忘次数提供精准复习建议，同时图拓扑定位知识盲区，不只是"这个知识点弱"，而是"因为前置节点未掌握，导致后续节点理解困难"

**自进化能力**：
- **LLM 驱动的自适应指令**：自动识别学习风格（公式导向/视觉型/例子驱动/记忆技巧型）、认知状态（专注/焦虑/过载/积极），动态调整教学策略
- **跨会话记忆**：记得你的目标院校、偏好的学习风格、上次说过的困惑
- **Trajectory 数据收集**：每次对话轨迹自动记录，为 MUTAR 自进化优化做数据储备

**典型对话**：
- "你这周微积分正确率下降了，建议先复习洛必达法则的 3 个考点，再做一组中等难度的极限题"
- "你离考研还有 87 天，根据你的薄弱点分布，这是本周的优先级清单"
- "从你的学习数据看，极限这个知识点平均难度 6.5，记忆稳定性只有 3.2 天，建议增加复习频率"

**场景文案**：
- 首页引导：「小宇已经分析了你的诊断结果，点击开始第一次规划对话」
- 空状态：「你好，我是小宇。让我看看你的学习数据，帮你理清接下来该做什么。」
- 侧边栏：「小宇 · 学习教练」

### 2.2 工作台 — 教师教研引擎

**一句话**：自然语言驱动全流程。出题、查数据、管作业，一句话完成。

**完整描述**：
工作台是教师的唯一 AI 教研入口。用自然语言描述需求——"出 10 道微积分极限的客观题，难度 normal"——Agent 自主搜索知识点、调用出题管线、展示结果。说"入库"就存入题库，说"精修"就启动 ARC 四 Agent 对抗管线。

工作台通过意图路由按需调度 15 个工具：出题、查学生数据、管理作业、浏览资产、发送通知、邀请管理、学情分析。教师不需要学习功能菜单，直接对话即可。

**ARC 对抗管线**（不是"AI 生成 → 过滤"）：
- **Author** 生成候选题
- **Reviewer** 开启深度思考模式，从区分度、清晰度、知识覆盖度三维评分
- **AuthorRevise** 根据反馈逐条修改
- **Classifier** 做最终审计：答案正确性、认知层级、难度合规、知识标签

不达标？退回重来，最多 3 轮。最终产出的题目，质量不亚于资深教研团队的产出。

**嵌入架构**：
- 难度由教师指定，模型在约束下生成
- 所有有实际后果的操作——布置作业、发送通知、题目入库——入口只有一个：教师点击确认
- 不存在绕过教师确认的路径。这不是可选项，是硬约束。

**场景文案**：
- 空状态：「描述你的出题需求，我来处理剩下的」
- 快捷指令：「针对薄弱点出题」「出一套模拟卷」「自定义出题」
- 侧边栏：「工作台 · AI 教研引擎」

## 三、核心能力文案

### 3.1 Memorix-Field 自进化记忆引擎

**标题**：不是间隔重复，是记忆进化 × 知识图谱。

**描述**：
市面上所有刷题产品都在用 Anki 的 SM-2 或 FSRS 的静态参数。Memorix-Field 完全不同——它是一个统一的自进化引擎，同时做两件事：

- **Weibull 遗忘模型 + 在线随机梯度下降**：每做一道题，个人记忆参数就实时更新一次。不是离线批量训练，是持续在线的个体拟合。算法在学习你。
- **知识图谱拓扑分析**：知识点不是标签，是图结构中的节点。掌握度沿图边传播，图拓扑定位盲区——不只是"哪个知识点弱"，而是"因为哪个前置节点未掌握，导致后续哪些节点理解困难"。
- **统一决策**：记忆衰减状态 + 知识结构依赖，共同决定"下一步该学什么"。不是两套系统各自建议，是一个引擎同时考量。

比 FSRS v4.5 预测 RMSE 降低 13.7%，用户留存率提升 9.2%。

**短文案**：
- 「Memorix-Field 在你做题的同时学习你：记忆曲线 + 知识地图，一个引擎全搞定。」
- 「每做一道题，算法也在学习你。」

### 3.2 4-Agent ARC 对抗出题管线

**标题**：四个 AI 博弈，一套高质量考题。

**描述**：
单次 LLM 出题通过率约 60%。ARC 对抗管线通过率 85%+。差距不在模型能力，在**架构**。

Author 写题 → Reviewer 用最强模型 + 深度思考模式逐题审查 → 不达标退回 AuthorRevise 修改 → Classifier 做最终审计（答案正确性、认知层级、难度合规、知识标签）。最多 3 轮迭代，不达标不放行。

**短文案**：
- 「单次 AI 出题通过率 60%。对抗管线 85%+。差距在架构。」
- 「不是 AI 帮你出题，是四个 AI 博弈出题。」

### 3.3 Agent 统一运行时

**标题**：新增一个 Agent，只需写工具和提示词。

**描述**：
UniMind 的所有 AI Agent 共享一套运行时：Bot 模型定义角色 → ToolExecutor 提供工具 → chat_dispatch 处理对话 → call_ai_with_tools 驱动最多 5 轮自主工具调用。

两个 Agent（小宇 + 工作台）各启用意图路由器，按关键词预筛选工具子集，精准匹配对话意图。

新增一个 Agent，只需要：
1. 在 Bot 表创建一条记录
2. 写一个 ToolExecutor 子类
3. 定义系统提示词

不需要重建对话系统、不需要新建 API 端点、不需要改前端。这就是基础设施的意义。

**短文案**：
- 「我们在造操作系统，不是一个 App。」
- 「Agent 即服务。定义工具和提示词，即刻上线。」

### 3.4 嵌入架构：AI 在执行域，教师在判断域

**标题**：嵌入，不替代。这条线是硬约束。

**描述**：
教育 AI 产品面对的核心问题：人和工具的关系怎么摆。

我们画了一条硬线——**AI 做执行，教师做判断**。难度设定、选题决策、布置时机、教学调整属于教师。搜索知识点、生成题目、校验质量、汇总数据属于 AI。

这不是"人类在回路中"的产品理念。在架构层面，它是**硬约束**：
- 所有有实际后果的操作，入口只有一个：教师点击确认
- 代码层不存在绕过教师确认的路径
- 没有"全自动"开关。执行效率的提升不以让渡判断权为代价

长期效果：不同机构用同一个系统，半年后系统的行为会不一样——因为每个机构的题库、出题偏好、质量标准，随着教师反馈逐渐形成自己的方向。

**短文案**：
- 「AI 在判断之前停手。」
- 「不是让 AI 替代教师，是让 AI 把教师的判断力放大。」

### 3.5 四层个性化

**标题**：从冷启动到深度了解，四层递进的个性化。

**描述**：

| 层级 | 机制 | 作用 |
|------|------|------|
| L1 诊断测试 | 首次登录 10 道题快速评估 + Field 图谱初始化 | 冷启动：初始化知识掌握水平 |
| L2 Memorix-Field | 每次答题实时更新记忆参数 + 图谱掌握度 | 行为建模：精准预测遗忘临界点 + 知识盲区定位 |
| L3 Agent Memory | 对话中自动提取偏好和习惯 | 深度个性化：越用越懂你 |
| L4 自适应指令 | LLM 分析记忆生成教学策略 | 动态调整：根据认知状态和学习风格优化教学 |

四层叠加，每个学生都有一个独一无二的学习模型。

**自适应指令详情**：
- **学习风格识别**：公式导向型、视觉型、例子驱动型、记忆技巧型
- **认知状态感知**：专注、焦虑、过载、积极主动
- **动态策略调整**：焦虑学生先共情再给小步骤，过载学生精简内容只聚焦最紧急问题

### 3.6 ARCA 四层架构

**标题**：系统的骨骼。四层自下而上，定义系统长什么样。

**描述**：
ARCA 是 UniMind 的系统架构——不是某个功能模块，而是整个产品的骨骼结构。四层自下而上：

| 层 | 名称 | 职责 |
|----|------|------|
| L1 | 基础框架 | 对话引擎 · Bot 运行时 · 工具系统 · 意图路由 |
| L2 | 记忆系统 | Memorix‑Field 调度引擎 · 存储引擎 · 语义记忆 |
| L3 | 教育闭环 | 教 · 练 · 测 · 评，通过记忆系统获取数据 |
| L4 | 测评引擎 | 评分 · 错因分析 · 变式题匹配 · ARC 出题管线 |

核心原则：Memorix 是闭环的节拍器——不做"立刻纠错"，错题按间隔重复节奏自然出现。测评引擎负责诊断，记忆系统负责调度。

**短文案**：
- 「ARCA：从对话到测评，四层架构承载整个系统。」
- 「一个架构，四个层次。不变的是结构，进化的是能力。」

### 3.7 MUTAR 自进化

**标题**：系统的进化引擎。运行在 ARCA 之上，系统如何变得更好。

**定位**：MUTAR 不是架构，是运行在 ARCA 之上的自进化引擎。ARCA 是骨骼（静态结构），MUTAR 是新陈代谢（动态进化）。

**描述**：
MUTAR（Measure→Umpire→Think→Adapt→Refine）是 UniMind 的自进化引擎。每次 Agent 对话自动采集轨迹，用户反馈（赞/踩）闭环标注，每周自动分析生成优化建议。

不是静态的 Prompt 工程——系统在运行中自我改进：
- 轨迹自动采集：每次对话的工具调用链、轮次数、成功率
- 用户反馈闭环：赞/踩反馈自动关联到对应轨迹，覆盖自动评估结果
- 优化建议生成：周期性聚合分析，识别高频失败模式，生成 Prompt 优化方案

**短文案**：
- 「ARCA 是骨骼，MUTAR 是进化。」
- 「你的 AI 在学习如何更好地教你。」
- 「不是调参，是进化。」
