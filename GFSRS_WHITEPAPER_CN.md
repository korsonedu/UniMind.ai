# UniMind 记忆调度引擎白皮书

## 从 FSRS v4.5 到 Memorix 到 GFSRS-Pro：自进化间隔重复算法体系

**版本**：v2.0  
**日期**：2026-05-08  
**作者**：UniMind Research

---

## 概述

间隔重复（Spaced Repetition）是学习科学中最具实证基础的高效记忆方法。其核心思想简单而深刻：在遗忘即将发生的临界点进行复习，以最小的复习次数实现最长的记忆保持。然而，将这一思想落地为可大规模部署的工程系统，需要解决三个核心挑战：

1. **个体差异**：不同学习者的遗忘速度差异巨大——有人过目不忘，有人反复遗忘。同一个调度参数不可能适配所有人。
2. **目标异构**：日常稳扎稳打与考前冲刺需要完全不同的复习策略——前者追求长期保持率，后者追求短期覆盖率。
3. **资源约束**：学习者每天的学习时间是有限的。系统必须像基金经理一样，在众多待复习项目中做出最优配置决策——哪些题目今天必须复习，哪些可以明天再说。

UniMind 的记忆调度引擎，是一套从经典 FSRS 出发、逐步演进的算法体系。本文档完整阐述其三层架构：**FSRS v4.5（基础预测层）→ Memorix（在线自进化层）→ GFSRS-Pro（决策优化层）**，以及在 UniMind 生产系统中的工程实践。

## 第一层：FSRS v4.5 — 记忆状态的数学建模

### 核心概念

FSRS（Free Spaced Repetition Scheduler）将人类的记忆状态抽象为三个潜变量：

| 变量 | 符号 | 含义 | 范围 |
|------|------|------|------|
| **稳定性** | S | 记忆从「完全掌握」衰减到 90% 保持率所需的时间 | (0, +∞) 天 |
| **难度** | D | 该知识点对特定学习者的固有难度 | [1, 10] |
| **可提取性** | R | 在特定时间点，学习者能正确回忆该知识的概率 | [0, 1] |

### 遗忘曲线

FSRS 假设人类的遗忘过程遵循双参数幂律衰减（power-law decay）。在最后一次复习后经过 τ 天后，可提取性 R 为：

```
R(τ) = (1 + α · τ / S)^(-γ)

其中：
  α = r₀^(-1/γ) - 1    —— 形状参数，使得 R(0) = r₀
  r₀                     —— 初始可提取性（≈ 0.98~1.0，表示刚学完几乎不会忘）
  γ                      —— 衰减速率系数
```

**直观理解**：S 越大（记忆越稳定），曲线下降越慢；γ 越大，对间隔时间越敏感（遗忘加速）。

### 状态转移方程

每次复习后，系统根据学习者的回答表现 g ∈ [1, 5]（1 = 完全忘记，5 = 轻松掌握），更新 S 和 D：

**成功回忆（g ≥ 3，y = 1）**——稳定性增长：

```
S' = S · [1 + exp(θ₁) · D^(-θ₂) · (S+1)^(-θ₃) · (exp(θ₄·(1-R)) - 1) · q(g)]

其中：
  q(g) = (g - 3) / 2    —— 将 1~5 的评分映射到[-1, 1]的修正区间
  θ₁, θ₂, θ₃, θ₄        —— 17 个可学习参数中的核心 4 个
```

- θ₁ 控制整体增长率
- θ₂ 控制难度对增长的抑制作用（越难的题，学会了稳定性涨得越多——反直觉但符合经验）
- θ₃ 控制当前稳定性的边际递减效应（已经非常稳定的记忆，再复习收益递减）
- θ₄ 控制「在快遗忘时才复习效果最好」这一间隔效应的强度

**失败回忆（g ＜ 3，y = 0）**——稳定性衰减：

```
S' = max(S_min, exp(φ₁) · D^(-φ₂) · (S+1)^(φ₃) · exp(φ₄·(1-R)))

其中 S_min = 0.1 天（遗忘后仍然保留极少量记忆痕迹）
```

**难度更新**：
```
D' = clip((1 - λ)·D + λ·D₀(g) + κ·(1-y) + ξ·(y - R), 1, 10)

其中：
  D₀(g) 按评分计算的标准难度
  λ     难度更新速率
  κ     失败时额外增加难度
  ξ     实际回忆结果与预测之间的差异修正
```

### 参数优化（离线批量学习）

FSRS v4.5 使用 L-BFGS-B 算法，在用户积累足够的复习日志（≥300 条）后，离线拟合 17 个个性化权重 w[0..16]。优化目标为最小化 RMSE（均方根误差）：

```
L(w) = √(1/n · Σ (R̂_i - R_i)²)

其中 R̂_i 是模型预测的可提取性，R_i 是实际观察结果
```

UniMind 的生产实现中，此过程通过 Celery Beat 每天凌晨 3:00 自动执行（`auto_tune_fsrs_for_all_users`），对 ReviewLog ≥ 300 条的用户逐一优化，生成 `FSRSProfile` 记录，并记录 `FSRSOptimizationLog` 追踪优化历史。

## 第二层：Memorix — 在线自进化记忆调度

FSRS v4.5 的离线批量优化存在固有缺陷：用户的遗忘曲线在持续漂移，但权重更新只在凌晨、且需要大量数据时才触发。这导致「刚考完期中进入期末冲刺」的用户，需要 1~2 周才能在参数层面感知到学习状态变化。

**Memorix** 是为解决这个问题而设计的在线学习层，核心创新有四项：

### 1. Weibull 遗忘模型

FSRS 的幂律衰减假设遗忘速率单调递减。但实际上，不同知识类型的遗忘规律截然不同：

- **突击记忆型知识**（如人名、年份）：初期遗忘极快（k < 1），Weibull 的危险率随时间递减
- **深度理解型知识**（如理论推导逻辑）：初期保持较好，但长时间不碰会加速遗忘（k > 1），危险率随时间递增

Memorix 用 Weibull 分布替代幂律：

```
R(t) = exp(-(t / λ)^k)

其中：
  λ = S（稳定性，尺度参数）
  k = w[16]（形状参数，域自适应）
```

k 本身就是可学习的参数，系统会自动为不同类型的知识域（货币银行学 vs. 衍生品定价）学习不同的 k 值。

### 2. 在线 SGD + Nesterov 动量

每次复习后立即更新权重，不等批量：

```
v_t = μ·v_{t-1} - η·∇L(w_t + μ·v_{t-1})    ← Nesterov 前瞻梯度
w_{t+1} = w_t + v_t                            ← 权重更新
w_{t+1} ← EMA(w_{t+1})                         ← 指数移动平均平滑
```

- Nesterov 动量（μ ≈ 0.9）提供惯性，防止单次噪声过度影响
- EMA 平滑提供短期稳定性，避免一次做错题就大改策略
- Brier Score（而非 RMSE）作为损失函数，对概率预测更敏感

### 3. 用户上下文化先验

新用户（冷启动）使用全局先验 θ_global，老用户拥有个性化权重 θ_u。更新规则带 L2 正则化向全局回归：

```
θ_u ← θ_u - η · ∇L(θ_u) + β · (θ_global - θ_u)
```

- β 控制向全局回归的强度（β 越大，越不相信个人的短期波动）
- 随着 ReviewLog 积累，β 自动衰减（系统越来越信任个人数据）

### 4. 后悔最小化调度

下一次复习时间的决策不再基于固定阈值（如 R < 0.9 就复习），而是基于期望后悔最小化：

```
R(t) = (1 - R̂(t)) + α · C(t)

其中：
  (1 - R̂(t))  期望遗忘损失
  C(t)         在该时刻复习的机会成本
  α            用户特定的成本敏感度
```

选 R(t) 最小的那个时间作为 `next_review_at`。对于会员用户（时间宝贵），α 更高——系统会优先保证他们在有限时间内只复习最有价值的题目。

### 工程标识

在 UniMind 中，通过环境变量控制 Memorix 开关：

```bash
USE_MEMORIX=true   # 启用 Memorix（关闭则回到 FSRS v4.5）
```

Memorix 权重为 20 维（FSRS 的 17 维 + Weibull k + 嵌入强度 + 成本敏感度），存储在 `FSRSProfile.weights` 中。

## 第三层：GFSRS-Pro — 分层贝叶斯与预算约束调度

GFSRS-Pro 是 Memorix 的进一步演进，面向「论文发表级方法学 + 生产系统核心」的双重目标。它在 Memorix 的基础上引入三项新的机制：

### 1. 分层贝叶斯个性化

```
Θ_u = μ_{c(u)} + W · h_u + ε_u
ε_u ~ N(0, Σ_{c(u)})
```

- c(u)：用户所属的群组（如「零基础跨考」「金融本科」「在职备考」）
- μ_c：群组先验均值
- W · h_u：行为嵌入向量 h_u 的线性投影
- Σ_c：群组内方差

该结构同时解决冷启动（新用户先用群组先验）和渐进适应（随着数据增多，后验逐渐向个人偏移）。

### 2. 预算约束下的 0-1 背包调度

将每日的题目选择建模为组合优化问题。每道题 i 对用户 u 在时刻 t 的边际效用为：

```
U_{u,i,t} = w₁ · ΔR̂_{u,i,t}  +  w₂ · ΔE^{exam}_{u,i,t}  -  w₃ · ĉ_{u,i,t}
            └─ 记忆保持增益 ─┘  └─ 考试覆盖增益 ─┘  └─ 时间成本 ─┘
```

每日选题退化为 0-1 背包问题：

```
max  Σ z_i · U_i
s.t. Σ z_i · ĉ_i ≤ B(d)      ← 每日学习时间预算
     z_i ∈ {0, 1}
```

**业务含义**：
- 日常模式：w₁ 权重高（优先保持长期记忆）
- 冲刺模式：w₂ 权重高（优先覆盖更多考点）
- 会员模式：B(d) 更大（允许更多题目量）

### 3. 自进化策略生成

```
For each user group:
  1. 检测遗忘漂移信号 → 生成候选参数/策略变体
  2. 用离线回放评估（Doubly Robust 估计器）
  3. 低流量 A/B 在线验证
  4. 自动晋升/回滚（通过安全阈值）
```

回滚触发条件（任一触发即回滚）：
- 7 天保持率下降 > 5%
- 日均复习负担增加 > 30%
- 用户投诉率上升

## 系统架构

```
学习者交互 → FSRS/Memorix 更新 → ReviewLog 持久化
                                          ↓
                        ┌─────────────────┴──────────────────┐
                        ↓                                    ↓
              Memorix 在线更新                       离线批量优化
         (每次答题后即时 SGD)                   (每日凌晨 Celery Beat)
                        ↓                                    ↓
                FSRSProfile.weights                   FSRSOptimizationLog
                        ↓                                    ↓
              Memorix 调度器  ←──── UserKnowledgeState (掌握度)
                        ↓
              PersonalizedMockExam (个性化模考 PDF)
```

### 数据流

| 表 | 用途 | 更新频率 |
|----|------|---------|
| `ReviewLog` | 每次复习的原始记录（grade, elapsed_days, predicted_retrievability） | 实时写入 |
| `UserQuestionStatus` | 每题对每用户的状态（stability, difficulty, reps, lapses, next_review_at） | 每次答题后更新 |
| `FSRSProfile` | 每用户的 17/20 维权重的持久化 | 凌晨批量更新 / 在线增量更新 |
| `FSRSOptimizationLog` | 每次优化的审计记录 | 凌晨批量写入 |
| `UserKnowledgeState` | 每用户每知识点的掌握度分数 | 每 6 小时批量计算 |
| `KnowledgePointAnnotation` | 用户对知识点的标注状态 | 同步自 UserKnowledgeState |

## 实验与评估协议

### 预测指标

| 指标 | 公式 | 含义 |
|------|------|------|
| RMSE | √(Σ(R̂ - R)² / n) | 预测精确度 |
| Brier Score | Σ(R̂ - y)² / n | 概率校准质量 |
| AUC | — | 区分可回忆/不可回忆的能力 |
| Calibration Error | Σ|bin_avg - bin_pred| / n_bins | 预测是否"诚实" |

### 策略指标

| 指标 | 含义 |
|------|------|
| 7/14/30 天保持率 | 在指定天数后正确回忆的比例 |
| 保持率/分钟 | 每投入一分钟复习的保持率提升 |
| 日间负担方差 | 每日复习量的波动程度（越低越好） |
| 考试关键覆盖率 | 高频考点中保持率 > 0.9 的比例 |

### UniMind 生产基线（内部跟踪）

系统已上线 FSRS v4.5 + Memorix 双轨运行，`USE_MEMORIX=true` 的用户占比可在 `FSRSOptimizationLog` 中按 `note` 字段分组统计。核心仪表盘路径：

- Django Admin → `FSRSOptimizationLog`（按 `accepted` 筛选看优化接受率）
- `/api/quizzes/fsrs/curve/`（校准曲线，返回分桶 RMSE/MAE）
- `/api/quizzes/fsrs/optimization-history/`（优化历史日志）
- `/api/graph/user_heatmap/`（知识图谱掌握度热力图）
