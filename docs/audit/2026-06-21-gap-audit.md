## 三、"基础设施"视角的核心缺失

定位是"教育机构基础设施"，对比真正的 infra 产品应有能力：

| Infra 能力 | 当前 | 差距 |
|-----------|------|------|
| **可观测性** | 仅 Django log + 超管 Dashboard | 无机构级用量仪表盘、无 Agent 调用成功率监控、无成本追踪 |
| **多租户隔离** | institution FK 全表覆盖 | 数据隔离完善，但**配额强制执行**依赖前端+权限检查，无硬限流 |
| **可扩展性** | 新增 Agent 三步法 | 好。但工具注册无版本管理，Bot prompt 热更新无 A/B 测试框架 |
| **自进化** | MUTAR T1-T7 | T8（反事实验证）缺失，无自动回滚机制 |
| **开放 API** | APIPlatform.tsx 占位 | 零实现。API Key/OAuth/计量/文档门户全无 |
| **企业部署** | Docker Compose + Systemd | 有文档。缺 K8s Helm Chart、模块开关（ENABLED_APPS）、飞书集成 |

## 五、正面：已建立的竞争壁垒

这些是真实的、已验证的优势，不是 PPT：

- **Memorix 调度**（Weibull + SGD + 再巩固窗口 + Field 图扩散）→ 国内领先
- **4-Agent ARC 出题管线**（对抗博弈 85%+ 通过率）→ 行业前沿
- **IRT 3PL 认知诊断**（MML-EM + 机构级参数隔离）→ 心理测量学标准
- **双 Agent 运行时**（意图路由 + 自适应 prompt + MUTAR 自进化）→ 架构先进
- **知识树体系**（12 学科 11908 节点 + 7 种边类型）→ 内容壁垒
- **6 天密集迭代速度** → 执行力壁垒

差距不在"有没有"，在"好不好用"。当前正在从"能跑"到"能用"的关键拐点。
