## 差距清单（按优先级排序）

### P0 — 阻塞机构客户签约（3 项）

| # | 差距 | 当前状态 | 为什么是 P0 |
|---|------|---------|------------|
| 1 | **企业版本地部署方案** | Enterprise 套餐写了 `brand.custom`，实际无本地部署能力 | 企业客户的核心诉求是数据在自己服务器、可二次开发（类似开源模式）。需提供：模块化代码结构 + 部署文档 + 配置管理 + 二次开发指南。飞书集成也放在企业版。 |
| 2 | **课程管理（机构端）** | 课程 CRUD 存在，但机构管理员无法将课程分配到班级/设置学习顺序 | 培训机构运营的基础——知道"这个班这学期上哪些课"。不做排课引擎，只需课程→班级关联 + 顺序编排。 |
| 3 | **班级成绩册 (Gradebook)** | 有班级 CRUD 和作业，无聚合成绩视图 | 教师日常最高频操作。班级-学生-考试三维成绩是机构采购决策的核心检查项。 |

### Phase 1：扫清签约障碍（2-3 个月）

**目标：让机构能用起来**

```

班级成绩册 ─────────── 班级-学生-作业三维视图 + 平均分/及格率

#### L1 课程编排详细设计

当前 Class 和 Course 完全独立——共享 `institution` FK 但无直接关联。本方案实现机构管理员将课程分配给班级、设置学习顺序。

**模型：`ClassCourse`**（`users/models.py`）

```
ClassCourse:
  class_obj   FK → Class      (on_delete=CASCADE)
  course      FK → Course      (on_delete=CASCADE)
  order       PositiveInt      (学习顺序，越小越前)
  publish_at  DateTime         (定时发布，null=立即可见)
  created_at  DateTime         (auto_now_add)

  Meta: unique_together('class_obj', 'course'), ordering=['order', '-created_at']
```

> 不用 M2M——无法携带 order/publish_at。不用 `due_at`——截止日期是作业层的概念。

**API：2 个新 View**（`users/views_institution.py`）

| 端点 | View | 功能 |
|------|------|------|
| `POST/GET /institution/me/classes/<id>/courses/` | `ClassCourseManageView` | 管理员批量设置课程（先删后建，事务内）+ 查询已分配课程 |
| `GET /me/class-courses/` | `StudentClassCourseView` | 学生获取本班课程列表（自动取所在班级，过滤 publish_at，附带 VideoProgress） |

另在 `CourseListCreateView.get_queryset` 增加 `class_id` 过滤参数（~5 行）。

**前端：2 处改动**

1. `ClassSection.tsx` — 每个班级行增加"课程"按钮 → 弹出课程编排面板（左侧机构课程库搜索+多选 → 右侧已分配列表拖拽排序 → 保存提交）
2. `CourseCenter.tsx` — 学生端顶部增加班级选择器，选班级后仅显示本班课程

**改动文件**：`users/models.py` + `users/serializers_institution.py` + `users/views_institution.py` + `users/urls.py` + `courses/views.py` + `ClassSection.tsx` + `CourseCenter.tsx` | **预估** ~300 行后端 + ~150 行前端

## 核心竞争力分析

### 不该做的（或现阶段不该做的）

| 事项 | 理由 |
|------|------|
| 在线课堂/直播 | **已明确不做**。腾讯会议/ClassIn 已解决，集成优于自建。 |
| 排课/课表引擎 | **已明确不做**。改为机构上传课程+班级关联即可，不做教室/场地/冲突检测/周视图课表。 |
| OCR 试卷批改 | 技术难度大、准确率瓶颈、不如推在线作答。 |
| 自建支付网关 | 已有 5 网关抽象够用。做支付不合规风险大。 |
| LTI/IMS 标准集成 | 国内需求极低。出海再说。 |
| 打字机逐字输出 | **暂缓**。当前一次性渲染可接受，等核心功能补齐后再优化。 |

### 应该聚焦的差异化高地

| 方向 | 现有基础 | 为什么能赢 |
|------|---------|------------|
| **AI 批改** | grading_engine + 主观题采分点 + Agent 工具 | 教师最大痛点。已有技术储备，是天然延伸。 |
| **AI 备课** | AI Agent + prompt 体系 + 知识树 | 教案/课件/课堂活动 AI 一键生成，传统 LMS 完全没有。 |
| **AI 学情诊断** | Memorix + ELO + IRT + UserProfile | 从"学得怎么样"到"接下来该学什么"的智能推荐，比静态报表强一个代际。 |
| **自适应学习路径** | 知识树 + 知识边（prerequisite）+ Memorix | 每个学生走不同的学习路径，是 AI-native 产品的终极形态。 |

### 与竞品的关键差异定位

不要做"又一个 ClassIn/校宝"。UniMind 的定位应该是：

> **AI-native 教育机构操作系统** —— 传统 LMS 管"教了什么"，UniMind 管"学会了没"。

具体差异化：
- ClassIn 做课堂交互 → UniMind 做课前备课+课后练习+学情诊断
- 校宝做教务管理 → UniMind 做 AI 驱动的教学质量管理
- 传统 LMS 是记录系统 → UniMind 是决策系统

### 企业版本地部署方案

企业客户不走 SaaS 多租户路线，而是**本地部署+二次开发**（类似开源模式）。这需要：

**现有基础：**
- Docker Compose + Systemd 部署已可用
- 配置通过环境变量管理（`.env`）
- 代码模块化程度尚可（11 Django apps）

**需补齐（Phase 1）：**
- **代码模块拆分**：企业可能只需要部分功能（如不要支付/不要自习室）。需支持按模块开关——`ENABLED_APPS` 配置或 Django app 级 feature flag。
- **部署文档**：从"开发者 README"升级为"运维部署手册"——含 Docker Compose 生产配置、Nginx SSL 终止、PostgreSQL 备份策略、Redis 安全配置。
- **品牌配置入口**：`Institution` 模型的 `logo`/`name`/`custom_domain` 字段已存在，需增加前端机构品牌设置页（Logo 上传 + 名称 + 主题色），部署时通过环境变量设默认值。
- **二次开发指南**：API 文档（已有 Swagger）、数据库 ER 图、关键服务调用链说明、Agent 扩展开发指引（新增 Bot 三步法已有雏形）。
- **飞书集成**：企业版专属模块——组织架构同步 + 消息通知 + 审批流，SaaS 版不包含。

**与 SaaS 版的关系：**
- SaaS 版 = 快速上手，按学员/用量付费
- 企业版 = 源码交付，客户自行部署运维，可选付费技术支持
- 核心引擎（Memorix、AI 管线、Agent 运行时）两版共享同一代码库

### 学习数据深度分析

UniMind 在学习科学基础设施上处于国内领先水平：

**世界级的：**
- Weibull 遗忘曲线 + 在线 SGD + Nesterov 动量 → 参数自适应
- 再巩固窗口理论（Nader 2000）工程实现 → 最优复习时机
- 知识图扩散调度（Memorix-Field） → 跨知识点复习收益
- IRT 3PL + Brier Score + 后悔最小化调度

**但关键 gap 在"决策层"：**

```
数据采集（A+）→ 状态建模（A）→ 决策调度（B）→ 学习路径生成（D）→ 用户感知（C-）
                                    ↑                      ↑              ↑
                              所有 gap 在这里          路径引擎缺失    无 Dashboard
```

具体来说：
1. **IRT theta** → 学生能力估计精准，但**未被用于自适应难度选择**
2. **KnowledgeEdge** → 知识点前置/相似/混淆关系完备，但**未被用于学习路径拓扑排序**
3. **UserProfile 6 维画像** → 学习风格已分析，但**仅影响 prompt 措辞，不影响调度策略**
4. **Memorix stability** → 题目级遗忘参数精准，但**未定义知识点级掌握标准**

### 移动端体验总结

**当前产品更适合：桌面浏览器 / Tauri 桌面端**
**不适合：手机浏览器作为主力学习工具**

移动端差距三要素：
1. **无 PWA** → 不能添加到主屏幕 → 用户每次要手动输入 URL → 留存归零
2. **无离线** → 地铁/弱网环境白屏 → 碎片化学习场景全丢
3. **无推送** → 复习提醒/作业截止无法触达 → 产品失去"教练"角色
