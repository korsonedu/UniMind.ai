# 考前个性化 PDF 试卷生成 (Printable Personalized Mock Exams)

## 1. 背景与动机
考研是一场纸笔考试。长时间的在线刷题会导致学生“提笔忘字”或缺乏大段时间的专注力。
在冲刺阶段（考前 1-2 个月），系统应提供将数字化错题和薄弱点转化为**实体纸质试卷**的能力。这不仅是一个极具仪式感的学习环节，也是一项非常容易变现的增值服务。

## 2. 核心架构设计

### 2.1 组卷算法 (Paper Generation Algorithm)
组卷不是简单的随机抽题，而是基于用户的专属数据池进行定向抽取：
1. **数据源**：
   - 用户的 Memorix 留存率极低的题目。
   - 用户在知识图谱中处于“红灯（薄弱）”状态的节点对应的经典真题。
   - 历史错题本中的“顽固错题”（连续做错 2 次以上）。
2. **结构控制**：
   - 必须符合 431 考研的真实题型比例（例如：选择题 30 分，名词解释 30 分，计算题 40 分，论述题 50 分）。
   - 算法需要在用户的弱点池中，按照上述比例填充题目。如果弱点池中某类题型不足，则从全局优质题库中按相似难度替补。

### 2.2 PDF 渲染引擎 (PDF Rendering Engine)
我们将采用 Python 后端的渲染技术栈：
- **方案 A (HTML to PDF)**：
  - 使用 Django 模板渲染一个包含 LaTeX 公式 (通过 MathJax/KaTeX 预处理) 的 HTML 页面。
  - 使用 `WeasyPrint` 或 `wkhtmltopdf` 将 HTML 转换为高清晰度的 A4 尺寸 PDF。
- **方案 B (原生 LaTeX 编译)**：
  - 使用 Jinja2 渲染一个 `.tex` 文件。
  - 在后台调用系统的 `xelatex` 命令，编译生成极其排版精美的 PDF。这是最专业的方案，尤其适合数学和金融公式密集的 431 考试。

## 3. 用户体验流 (User Workflow)

1. **发起请求**：用户点击“生成本周专属模拟卷”。
2. **生成与下载**：系统后台异步生成两份文件：
   - `[张三] 431冲刺模拟卷_试卷版.pdf` (留白用于作答)。
   - `[张三] 431冲刺模拟卷_解析版.pdf` (包含极其详细的解答和知识点溯源)。
3. **闭环**：鼓励用户在线下做完后，对着解析版自己批改（或拍照上传调用前面的 AI 主观题阅卷引擎），然后在系统中将做错的题标记为 `Again`，重新注入 Memorix 循环。

## 4. 后续开发指南
- 后端需要引入 `WeasyPrint` 或 `subprocess` 调用 `xelatex`。
- 考虑到 PDF 生成耗时较长（可能 10-30 秒），必须使用 Celery 异步处理，生成完毕后通过 WebSocket 或站内信/邮件通知用户下载。
