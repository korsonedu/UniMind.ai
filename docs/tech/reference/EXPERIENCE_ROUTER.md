# 经验路由器：UniMind 专属自进化引擎

> 设计日期：2026-06-13
> 核心命题：比 GEPA 更好的不是优化算法，而是经验的多维度路由

---

## 一、GEPA 的根本问题

### 1.1 GEPA 做了什么

```
轨迹 → LLM 反思（"哪里不好"）→ 改 prompt → 测试 → 好的留下
```

GEPA 本质上只做一件事：**把经验变成 prompt**。

### 1.2 它跳过了最关键的一步

从「失败在哪里」到「改什么」，GEPA 中间没有**提取可复用规律**这一步。

反思是一次性的、用后即弃的。它不区分：

- 这个失败是张三的个别问题，还是所有学生的共性问题？
- 这个问题是 prompt 策略错了，还是工具能力不够？
- 这条规律的适用范围是什么？有效期多长？

因为没有这层判断，所有经验只能灌进一个地方——prompt。

### 1.3 为什么不是「多维度同时优化」

直觉方案：把 GEPA 扩展到四个维度，各跑各的优化器。
结论：不行。因为四个维度不是独立的——**把本该写入 memory 的规律写进 prompt，会对所有学生产生噪音。**

问题不是「优化不够强」，而是「规律没有去对地方」。

## 二、核心突破：经验路由器

### 2.1 一句话定义

**经验路由器（Experience Router）是从 Agent 对话轨迹中提取可复用规律，并判断每条规律应该写入哪个维度的路由系统。**

### 2.2 三层结构

```
第一层：反思（Reflection）
  输入：对话轨迹（消息、工具调用、输出、学生后续正确率变化）
  输出：失败/成功在哪里（"这里做得不好"）
  角色：GEPA 已有的能力，我们保留

第二层：提取（Extraction）  ← GEPA 缺失的
  输入：轨迹 + 反思
  输出：可复用的规律（"从这个案例中学到什么"）
  关键：规律 ≠ 修复方案。规律是陈述性的、有范围的、可验证的

第三层：路由（Routing）
  输入：规律
  判断：
    ├─ 这条规律对谁有效？（张三 / 全班 / 所有学生）
    ├─ 这条规律的生命周期？（一次性的 / 该 KP 范围内 / 永久）
    ├─ 这条规律的本质是什么？（教学策略 / 工具参数 / 协作顺序 / 个体记忆）
  输出：路由目标 + 写入粒度
```

### 2.3 同一个轨迹，不同性质的规律

举例：小宇给张三讲因式分解，用了图形化方法，张三从 60% 提升到 85%。

```
轨迹 → 反思：图形化讲解效果好

提取出的规律（同一轨迹可能产生多条）：

规律1：张三对几何直觉敏感，代数推导容易混淆符号
  → 路由：memory（个体级别，只对张三生效）
  → 形式：写入 AgentMemory，标签：geometry_intuition, 张三

规律2：因式分解用图形化讲解比代数推导效果好
  → 路由：prompt（教学策略段，全局适用）
  → 形式：写入 system_prompt 的教学策略部分

规律3：先讲解图形再出题比直接出题效果好，学生正确率高 12%
  → 路由：workflow（对因式分解及其下游 KP 生效）
  → 形式：对依赖链上的 KP，默认采用「先讲后练」顺序

规律4：图形生成工具在多项式次数 > 3 时产出速度慢
  → 路由：tool（参数级）
  → 形式：复杂多项式 → 降级为代数推导（不是图形差，是渲染慢）
```

## 三、四维度路由规则

### 3.1 Prompt 维度

**写入条件**：规律对所有学生有效、不依赖特定个体特征、属于教学策略层面。

**写入粒度**：段落级 patch，不是整段替换。改「教学策略」段不改「人格设定」段。

**示例**：
- 「因式分解优先用图形化引导」→ 写入 system_prompt 教学策略段
- 「学生说不会时先反问而非直接给答案」→ 写入 personality 段

### 3.2 Memory 维度

**写入条件**：规律限定于特定学生、或特定学生群体的特征。

**写入粒度**：按学生 ID + KP 标签存储，设置 TTL（如 30 天未触发则降权）。

**示例**：
- 「张三符号处理能力弱」→ 写入 AgentMemory，下次出代数题时自动降低依赖符号处理的题型比例
- 「李四对应用题有畏难情绪」→ 写入 mem0 语义记忆，出题时自动加一句鼓励

### 3.3 Tool 维度

**写入条件**：规律涉及工具的能力边界或参数配置。

**参数级**（可自动）：工具暴露可调接口的参数。
**能力级**（不可自动）：工具缺少某种能力，需开发。

**示例**：
- 参数级：「知识树检索 topK 从 5 改为 3，减少无关 KP 干扰」→ 自动调整
- 能力级：「学生输入'二次函数图像'搜不到'二次函数的图象与性质'，需要模糊匹配」→ 生成报告 + 人工

### 3.4 Workflow 维度

**写入条件**：规律涉及 Agent 协作顺序、模式选择、或步骤增删。

**写入粒度**：按 KP 类别或依赖链范围设置。

**示例**：
- 「因式分解类 KP，小宇先讲解 → 工作台出题」→ 对该类 KP 的默认 workflow
- 「简单算术题跳过工作台，小宇独自完成」→ 模式 D 触发条件

## 四、Memorix 依赖图：规律验证引擎

### 4.1 GEPA 的验证方式

GEPA 改完 prompt → 跑测试集 → 看 LM-as-Judge 分数。问题是：

- LM-as-Judge 只是 proxy，和真实效果有 gap
- 验证的是整个 prompt 的效果，无法归因到单条规律

### 4.2 我们的验证方式

**用学生后续做题的正确率变化，沿依赖图做反事实验证。**

```
一条规律声称：「图形化讲解对因式分解好」

验证步骤：
1. 在依赖图上找到「因式分解」的下游 KP（二次函数、多项式运算...）
2. 对比两组学生：
   - 应用了这条规律的学生（实验组）
   - 同水平但未应用这条规律的学生（对照组）
3. 看两组在下游 KP 上的正确率差异
4. 如果差异显著 → 规律验证通过 → 提升权重
5. 如果无差异 → 规律可能是偶然 → 标记待观察
```

### 4.2 为什么依赖图是关键

验证「图形化讲解对因式分解好」，不能只看下次做因式分解的正确率——那可能是练习效应。要看**这条讲解是否帮助学生更好地掌握了因式分解的下游 KP**。

Memorix 已经建好了 8360 KP 之间的前驱依赖边。这些边提供的是因果检验的基本单元：A 是 B 的前驱，A 学好没有，应该反映在 B 的成绩上。

**这是其他系统没有的。GEPA 永远只能在一个 benchmark 上打分，无法追踪知识沿依赖图的迁移效果。**

## 五、学生成绩是 ground truth

### 5.1 为什么比 LLM 评分硬

GEPA 的评估：LLM 判断「这段对话好不好」。

我们的评估：**学生下次做同类题的正确率变化。**

LLM 评分是主观的、有噪声的、可能与真实学习效果无关的。
学生成绩是客观的、有因果含义的、说明这轮对话真正产生了学习效果。

这不意味着抛弃 LLM 评估——LLM 评估可以用于快速初筛（哪些对话可能值得深挖）。但最终验证信号必须是学生成绩。

### 5.2 信号来源

- 做题正确率（quizzes app）
- Memorix 掌握度变化（Δmastery）
- 学生赞/踩反馈（已接入 AITrajectory.outcome）
- 教师标记（未来）

### 5.3 信号时效

成绩变化需要时间窗口。一条规律从应用到观测到效果可能需要 1-7 天（取决于学生有没有做下游 KP 的题）。这要求经验路由器支持**延迟验证**——规律先标记为 pending，后续做题数据触发验证。

## 六、与现有 MUTAR 管线的对接

### 6.1 现有管线（保留）

```
对话结束 → record_trajectory_async → AITrajectory 表
    → _auto_evaluate_trajectory（启发式评估）
    → 用户反馈覆盖
    → 每周 analyze_trajectory_task → Redis mutar:suggestions
```

### 6.2 新增层（经验路由器插在中间）

```
对话结束 → record_trajectory_async → AITrajectory 表
    → _auto_evaluate_trajectory（保留，快速初筛）
    ↓ 筛选：outcome=success 且 Δmastery>阈值 的轨迹
    ↓
[新] experience_extract_task（Celery，异步）
    → 从成功轨迹提取规律
    → 判断规律性质（对谁有效 / 生命周期 / 本质维度）
    → 路由写入：
        ├─ memory: AgentMemory + mem0
        ├─ prompt: mutar_variants.py（生成新 variant）
        ├─ tool: 参数调整 或 生成能力缺口报告
        └─ workflow: workflow 配置表
    ↓
[新] experience_verify_task（延迟触发，学生做题后）
    → 沿依赖图验证规律因果有效性
    → 更新规律权重 / 标记失效
    ↓
每周 analyze_trajectory_task（保留，增加规律统计维度）
```

### 6.3 复用现有基础设施

| 现有组件 | 复用方式 |
|---------|---------|
| `record_trajectory_async` | 轨迹采集不变 |
| `AITrajectory` 表 | 新增字段 tracking experience_id |
| `mutar_variants.py` | variant 分流 → 规律触发的 prompt 变体 |
| `AgentMemory` + `mem0` | memory 维度的写入目标 |
| Memorix α 调整 | tool 维度的参数级调整示例 |
| `analyze_trajectory_task` | 保留，增加规律效果统计 |

## 七、优化的方式

GEPA 的优化方式：遗传算法随机变异 → Pareto frontier 保留互补 → 整段替换 prompt。

我们的规律不是"整段 prompt 候选"，而是更细粒度的经验。不能套同一套。

### 7.1 规律的生命周期

每条规律从提取到持久化，经历三个置信度阶段：

```
刚提取 → confidence=low（建议）
  ↓ 同一场景再次触发，效果正面
小范围验证通过 → confidence=medium（验证）
  ↓ 依赖图反事实通过
大范围验证通过 → confidence=high（持久化）
  ↓ 长期验证稳定
沉淀为系统能力
```

**关键**：低置信度规律不污染全局。它只作为建议临时注入本轮对话，不写入任何持久层。

### 7.2 Prompt 维度：三阶段写入

Prompt 是全局生效的，写入必须保守。

```
阶段一（建议，confidence=low）
  形式：作为追加指令注入当前对话 context
  例子："[本次建议] 尝试用图形化引导讲解因式分解"
  条件：下 3 次同类对话中生效
  效果差 → 丢弃
  效果好 → 进入阶段二

阶段二（验证，confidence=medium）
  形式：生成 prompt variant，5% 流量分流
  对比：variant 组 vs baseline 组在下游 KP 的正确率差异
  显著优 → 进入阶段三
  无差异 → 降回阶段一或丢弃

阶段三（持久化，confidence=high）
  形式：段落级 patch 写入 system_prompt 对应段
  不是整段替换——只改"教学策略"段，不改"人格设定"段
  写入后 variant 退役（合并到 baseline）
```

**与 GEPA 变异的关键区别**：

- GEPA 变异是随机的，我们的规律是轨迹中提取的具体经验
- GEPA 直接整段替换 prompt，我们是段落级 patch
- GEPA 没有分阶段——改完直接跑测试集。我们有建议→验证→持久化三级漏斗

### 7.3 Memory 维度：触发条件 + 检索优先级

Memory 不是存了就算——规律必须定义触发条件，否则写入后永远不被检索。

```
规律：「张三符号处理能力弱」

写入形式：
  target: student=张三
  trigger: 出题涉及代数运算 OR 判分发现符号错误
  effect: 降低该学生对符号依赖型题目的比例 20%

生效方式：
  不是被动检索（等着 mem0 碰巧搜到）
  而是主动拦截——当小宇/工作台准备给张三出代数题时，
  在工具调用的预检层检查触发条件，命中则注入
```

**触发条件语法**：

```
trigger:
  event: 出题 | 讲解 | 答疑 | 搜索知识树
  scope: student=<ID> | kp=<KP_ID> | kp_chain=<KP_ID>
  condition: <表达式>
```

**记忆冲突处理**：同一场景触发多条 memory 规律 → 按权重排序，取 top 3 注入 context，不覆盖。

### 7.4 Tool 维度：参数调整 + 自动回滚

参数级：自动调整，强制回滚边。

```
规律：「topK=5 检索到太多无关 KP」

调整：
  knowledge_tree_search.topK: 5 → 3
  记录：调整时间、原值、调整原因、关联 experience_id

监控：
  调整后 7 天内，该工具调用的后续正确率变化
  如果变差 → 自动回滚到 5
  如果无变化 → 继续观察 7 天
  如果变好 → 确认，标记为 stable
```

能力级：生成缺口报告。报告中包含：
- 能力缺口描述
- 影响的 KP 范围和严重程度
- 建议的实现方式
- 关联的轨迹 ID（可追溯）

### 7.5 Workflow 维度：条件规则，收敛到基模式

Workflow 不能无限碎片化——每个 KP 都有特殊规则会让系统行为不可预测。

**收敛到 3-5 种基础模式**：

| 模式 | 适用 |
|------|------|
| 模式A：先讲后练 | 新知识、依赖关系复杂 |
| 模式B：直接出题 | 复习、简单KP |
| 模式C：先测再补 | 诊断不明确 |
| 模式D：单Agent快速 | 简单算术、信息查询 |

规律不创建新模式，而是**调整模式的选择条件和参数**：

```
规律：「因式分解类 KP 用先讲后练，但讲解时间不超过 3 步」

实现：
  mode_selector 在匹配 KP 类型时：
    基础匹配 → 模式A（先讲后练）
    叠加规律 → 讲解步骤上限=3
```

### 7.6 多条规律的冲突与组合

同一场景可能触发多条规律。冲突处理策略：

**同类规律（同维度）**：权重投票，取 top 3。
- 权重 = 验证次数 × 最近一次验证的 DMastery（下游 KP 掌握度变化）

**跨维度规律**：不冲突，组合注入。
- Memory（个体限定）+ prompt（全局策略）+ workflow（模式选择）可以共存
- 顺序：workflow 先决定模式 → prompt 注入策略 → memory 注入个体限制

**矛盾规律**：生成矛盾报告，等待人工裁决。不应自动选择。

### 7.7 规律的衰退与淘汰

规律不是永久的。

| 条件 | 动作 |
|------|------|
| 30天内未触发 | 权重衰减 50% |
| 60天内未触发 | 标记为 dormant，不再注入 |
| 90天内未触发 | 归档，保留记录但不生效 |
| 任意阶段验证失败 | 立即降权，记录失败原因 |
| 3次验证失败 | 永久淘汰 |

目的：防止记忆膨胀、防止过时的策略继续生效。

## 八、与 GEPA 的本质区别

| 维度 | GEPA | 经验路由器 |
|------|------|-----------|
| 核心问题 | 怎么改进 prompt | 规律该去哪 |
| 中间层 | 无（轨迹→修复直接跳） | 提取→路由→验证 |
| 输出 | 新 prompt 文本 | 规律写入 prompt/memory/tool/workflow |
| 验证信号 | LLM-as-Judge | 学生下游 KP 成绩 |
| 验证机制 | 测试集分数 | 依赖图反事实 |
| 优化范围 | 单 prompt | 四维度协同 |
| 粒度 | 整段替换 | 段落 patch / 条目 / 参数 / 拓扑 |
| 壁垒 | 任何系统都能用 | 依赖 Memorix 图 + 学生成绩，仅 UniMind 有 |

## 九、落地路径

### 第一阶段：规律提取 + 路由（1-2 月）

**目标**：让经验路由器能工作——从成功轨迹提取规律，正确路由到 memory 和 prompt。

**具体任务**：
1. 实现 `experience_extract_task`：LLM 从轨迹提取规律（结构化 JSON 输出）
2. 实现路由判断逻辑：规律类型 → 目标维度 + 粒度
3. 先开通 memory 和 prompt 两个维度（技术风险最低）
4. 人工抽检：路由判断是否准确（初期人工确认，积累信心后自动化）

### 第二阶段：依赖图验证（2-3 月）

**目标**：规律不再是写了就算，而是能被验证。

**具体任务**：
1. 实现延迟验证触发：学生做下游 KP 题 → 触发对应规律的验证
2. 实现反事实对比：实验组 vs 对照组在下游 KP 的正确率差异
3. 规律权重更新：验证通过 → 提升权重 → 更频繁应用
4. 规律失效标记：验证失败 → 降权 → 生成复盘

### 第三阶段：tool + workflow 维度（3-4 月）

**目标**：四个维度全部跑通。

**具体任务**：
1. tool 参数级自动调整接口（参考 Memorix α 调整模式）
2. tool 能力级报告生成
3. workflow 配置表的规律驱动更新
4. 四维度联动效果评估

## 十、为什么这是壁垒

1. **Memorix 依赖图是因果骨架**。GEPA 随机变异，我们沿依赖边定向优化。KP 之间的前驱关系不是说有就有的——需要 12 个学科的完整知识树 + 6758 条边 + 语义边。重建成本极高。

2. **学生成绩是 ground truth**。任何不做教育场景的系统只能用 LM-as-Judge 评分——那是 proxy。我们有真实的做题数据，信号干净。

3. **经验路由是维度乘法**。单维度优化效果加法级，四维度协同是乘法级。但协同的前提是路由对——错误的协同互相干扰。路由判断依赖 Memroix 图结构，不是通用能力。

4. **数据飞轮**。路由越准 → 规律越有效 → 学生成绩越好 → 更多高质量轨迹 → 更多可提取的规律。这个飞轮的起点（教育数据）只有做教育产品的公司有。
