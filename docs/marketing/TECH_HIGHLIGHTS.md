## 1. Memorix 自进化记忆调度算法

市面上几乎所有刷题产品都在用 Anki 的 SM-2 算法或其变体。我们自研了 **Memorix**——基于 Weibull 遗忘曲线 + 在线随机梯度下降 + 遗憾最小化的下一代记忆调度引擎。

**与 FSRS/Anki 的本质区别：**
- **Weibull 遗忘曲线**：比 FSRS 的幂律近似更精准地拟合人类遗忘规律
- **在线 SGD**：每个答题行为都被实时纳入梯度更新，模型随用户进化，不需要离线批量训练
- **遗憾最小化调度**：不只考虑"该不该复习"，还考虑"复习哪道题对整体记忆收益最大"

**效果**：同等复习时间，知识点留存率比传统间隔重复算法提升显著。

## 2. 4-Agent ARC 对抗出题管线

单次 LLM 出题的质量不稳定是行业通病。我们实现了四角色对抗管线：

```
Author (deepseek-v4-pro) → Reviewer (deepseek-v4-pro + thinking + 知识库Agent) → AuthorRevise → Classifier
```

- **Reviewer 是唯一开深度思考的环节**：用最强模型 + thinking + 多轮 Agent 研究，评审前可自主查询知识点定义和已有题目库，检查知识覆盖度和雷同重复
- **对抗循环最多 3 轮**：审稿人驳回 → 作者修改 → 再审，质量不达标不放行
- **零脆弱 JSON 解析**：全线 `agentic_structured_output` + `structured_output`，模型通过 function calling 强制输出合法 JSON，彻底消灭正则提取，净删除 220+ 行脆弱的 JSON 修复代码
- **Reviewer 从三唯独评分**（区分度、表述清晰度、知识覆盖度），任一维度 < 0.4 直接打回

**效果**：AI 出题准确率大幅提升，人工审核通过率显著提高。Reviewer 拥有知识库访问能力后，能精准判断题目是否命中目标知识点、是否与已有题目雷同。

## 3. 三层权限 + 多租户数据隔离

不是简单的 RBAC。我们的权限模型覆盖：

- **平台级**：超级管理员 → 运营 → 普通用户
- **机构级**：机构管理员 → 教师 → 学员，每个机构是独立租户
- **功能级**：基于订阅方案的功能门控 + 配额矩阵（总量型/月计型）

所有查询在数据库层做机构隔离，不存在"忘记加 filter 导致数据泄漏"的风险。

## 4. AI 模型细粒度路由

不做"全局一个模型"的粗糙配置。每个任务类型独立路由：

| 任务 | 模型 | Thinking | 工具 | 原因 |
|------|------|----------|------|------|
| 出题 Author | deepseek-v4-pro | 关 | structured_output | 结构化输出足够 |
| 出题 Reviewer | deepseek-v4-pro | 开 | agentic_structured_output | 多轮 Agent 研究 + 深度思考，可查知识库 |
| 主观题判分 | deepseek-v4-pro | 关 | structured_output | 结构化 JSON 输出 |
| AI 助教对话 | deepseek-v4-flash | 关 | call_ai_with_tools | 多轮 Agent，可查知识树+错题+薄弱点 |
| 对话/面试 | deepseek-v4-flash | 关 | — | 快速流式响应 |
| 作文评分 | deepseek-v4-pro | 开 | — | 最高强度推理 |

支持环境变量全局覆盖 + 按任务覆盖，灵活度和成本控制兼备。

## 6. 工程基础设施

- **熔断器 + 指数退避**：AI API 故障自动降级，不会拖垮用户体验
- **请求 ID 全链路追踪**：每个请求带 UUID，从 nginx → Django → Celery → AI API 可追溯
- **加密字段存储**：支付密钥等敏感数据 Fernet AES 加密落库
- **Cookie Token 认证**：httpOnly cookie + Authorization header 双通道，防 XSS 窃取
- **分片上传 + 断点续传**：大视频上传可靠，支持并发分片和恢复
