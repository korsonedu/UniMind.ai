# 小红书营销方案 — UniMind.ai 国内 B2B 内容策略

> 核心差异：小红书 ≠ 朋友圈（太随意）≠ X（太技术/英文）。小红书是"专业的真诚"——看起来像分享，实际上每一篇都在为品牌蓄水。

---

## 一、平台特性与我们的机会

### 小红书 vs 其他平台

| 维度 | 朋友圈 | 小红书 | X |
|------|--------|--------|-----|
| 受众 | 已加好友，信任基础高 | 陌生人，靠内容建立信任 | 全球 EdTech 圈 |
| 内容生命周期 | 24 小时 | 3-6 个月（搜索流量持续） | 48 小时 |
| 内容调性 | 随意、真实 | 专业 + 真实 + 美观 | 简洁、观点驱动 |
| 分发逻辑 | 时间线 | 推荐算法 + 搜索 | 推荐算法 |
| 商业转化 | 私聊 → 成交 | 种草 → 私信 → 成交 | DM → Demo → 成交 |
| 我们的目的 | 维护已有关系 | **获取新客户认知** | 获取海外客户 |

### 为什么小红书对 UniMind 很重要

1. **搜索流量长尾**：培训机构老板搜索"培训机构管理系统""AI 出题系统""教务系统推荐"→ 你的帖子一年后还在。这是朋友圈做不到的。
2. **决策者在这里**：中小机构创始人/校长刷小红书找解决方案是常态。不是刷 X，不是刷朋友圈（朋友圈里没有解决方案）。
3. **信任建立效率高**：一篇深度干货 = 比 10 次电话推销更有效的信任建立。小红书用户习惯"看完一篇长文 → 觉得这个人靠谱 → 私信咨询"。
4. **竞品真空**：目前几乎没有 AI 教育 SaaS 在小红书做系统的 B2B 内容营销。你先占住关键词，后来者很难追上。

## 二、目标受众画像

### 主要受众：培训机构决策者

**画像：**
- 25-45 岁，中小培训机构创始人/校长/教务主管
- 痛点：老师效率低、内容生产成本高、规模化困难、学生流失
- 搜索行为：搜"培训机构怎么提效""教务系统哪个好用""AI 能不能帮出题"
- 内容偏好：干货（运营经验/工具测评）> 故事（创始人经历）> 产品介绍

**他们不是技术背景。** 不讲 Weibull 分布，讲"学生做错的题自动安排重做"。

### 次要受众：教育行业从业者

**画像：**
- 机构老师、课程研发、教育创业者
- 搜索行为：搜"AI 教育工具""智能题库""自适应学习"
- 内容偏好：行业趋势、工具测评、职业发展

### 潜在受众：考研/考证学生

**画像：**
- 正在备考的考生（431 金融、CPA、法考等）
- 他们不买系统，但他们会推荐给自己报的机构
- 内容偏好：学习方法、备考经验、好用工具推荐

## 三、内容支柱与占比

### 支柱 1：干货输出（40%）

**目的：** 建立专业信任，吸引搜索流量
**形式：** 图文干货帖（首图吸引点击 + 内页长文）

**选题方向：**
- 培训机构运营痛点与解法（"你的机构还在让老师手出试卷吗"）
- AI+教育行业分析（"AI 不会替代老师，但不用的老师会被用的替代"）
- 间隔重复/自适应学习科普（面向非技术读者）
- 教务管理效率提升方法论

### 支柱 2：产品种草（25%）

**目的：** 展示产品能力，埋下购买欲望
**形式：** 场景化展示（不要功能列表）、使用教程、效果对比

**选题方向：**
- 真实场景演示："一个机构老师的一天——用 AI 之前 vs 之后"
- 功能教程：不是说明书，是"3 分钟搞定一张模拟卷"
- 数据对比：Before/After（出题时间、学生留存、老师工作量）
- 桌面 App 防作弊演示——这个在小红书上没人发过，天然差异化

### 支柱 3：创始人故事（20%）

**目的：** 人格化品牌，建立情感连接
**形式：** 长文叙事、创业日记、真实感悟

**选题方向：**
- "我为什么从 to C 转做 to B"（与同行共鸣）
- "做教育 SaaS 这一年踩过的坑"
- "Memorix 算法的从 0 到 1"（技术叙事但讲人话）
- 客户 demo 翻车故事（真实感）

### 支柱 4：客户案例（15%）

**目的：** 社会证明，消除购买顾虑
**形式：** 数据对比图、客户采访整理、使用前后对比

**选题方向：**
- "某机构的 AI 出题实践：出题从 8 小时到 30 分钟"
- "用了 Memorix 自适应复习后，学生留存率提升 23%"
- 客户真实评价截图（打码）

## 四、帖子模板与示例

### 模板 A：干货长文帖（适合首图+多图轮播）

**结构：**
```
首图：痛点问题 + 解决方案预告（大字标题）
图 2-4：问题分析（为什么现状不好）
图 5-7：解决方案（怎么做）
图 8-9：工具推荐（自然引出 UniMind）
末图：CTA（私信/评论/进主页）
```

**示例 A1：培训机构出题效率**

首图标题：**你的老师 40% 的时间没在教学，而在做这件事**

内容要点：
- 培训机构老师的真实时间分配：出题 30% + 批改 25% + 备课 25% + 真正教学 20%
- 手工出题的 5 个坑：重复劳动、难度不可控、知识点覆盖不全、更新慢、老师累
- AI 出题的解法：不是"ChatGPT 生成一道题"，是"知识树 → 出题 → 审题 → 分类"的自动化管线
- 效果对比：某机构 431 金融题库，原来 3 个老师 2 周出 500 题，现在 1 个老师 2 天审核完 AI 出的 500 题
- 最后：你的机构还在手工出题吗？评论区聊聊你们一个月出多少题

**示例 A2：自适应复习科普**

首图标题：**为什么你的学生做题量很大，分数却不涨**

内容要点：
- 刷题 ≠ 提分。大量做题但做的是"已经会的"和"忘光了的"——两头浪费
- 间隔重复的核心逻辑：在你刚好快忘的时候把题推给你——这个时间点每个学生不一样
- 传统方法的问题：所有学生同一套题→会的反复做、不会的没重点→挫败→流失
- 自适应怎么做：系统记录每道题的掌握度→预判遗忘时间→精准推送
- 最后：你觉得现在的刷题模式能帮学生提分吗？

### 模板 B：场景对比帖（Before/After）

**结构：**
```
图 1：Before — 痛点场景（如：深夜还在手出试卷的老师）
图 2：After — 使用 UniMind 后的场景
图 3-5：具体对比数据
末图：CTA
```

**示例 B1：**

首图标题：**一个机构老师的工作日——用 AI 之前 vs 之后**

| | Before | After |
|---|--------|-------|
| 周一 | 手工出下周模拟卷，8 小时 | AI 生成初稿，30 分钟审核 |
| 周三 | 批改 200 份试卷，一整天 | AI 初判 + 老师复核，2 小时 |
| 周五 | 整理错题，Excel 手动统计 | 系统自动归类，一键导出 |
| 周末 | 加班做下周课件 | **正常休息** |

结尾：老师在系统上花的时间少了，在学生身上花的时间多了。这才是教育科技该做的事。

### 模板 C：创始人叙事帖

**结构：**
```
首图：个人工作照或产品截图（有人的照片 > 没人的照片）
正文：一个真实故事 → 一个感悟 → 一个开放性问题
```

**示例 C1：**

标题：**我做了一套 AI 出题系统，然后客户跟我说他最需要的不是它**

正文大意：
> 花了一年打磨 AI 出题管线。3 个 Agent 对抗，质量闭环，效果确实好。Demo 的时候特别自信。
>
> 一个校长看完说："题出得不错。但我最头疼的不是出题。"
>
> 我：？
>
> 他："学生在我系统里刷题，旁边开着百度搜答案。你能解决这个吗。"
>
> 我愣了几秒。网页端确实做不了防作弊。回去就开始搞桌面 App——锁屏、禁切窗口、禁截屏、进程检测。
>
> 然后 Demo 下一个客户的时候，演示完防作弊锁屏，对方主动问我什么时候能上线。
>
> 这件事教会我一个道理：你以为你在卖 AI 出题，客户想买的可能是考试诚信。同一个产品，不同角色看到的价值完全不同。
>
> 做 B2B 最大的坑，就是以为自己知道客户要什么。

### 模板 D：工具测评/推荐帖

**结构：**
```
首图：产品截图 + "测评"标签
正文：客观介绍 → 适用场景 → 优缺点 → 适合什么人
```

**示例 D1：**

标题：**培训机构选 AI 系统，看这 3 点就够了**

内容要点：
1. 看 AI 出题是不是"一次性"的——单次 LLM 调用还是多 Agent 审核？前者准确率 ~60%，后者 85%+
2. 看有没有自适应复习——不是简单的错题本，是能预测遗忘曲线、精准推题的调度系统
3. 看能不能管考试——在线考试 + PDF 试卷 + 防作弊 + 离线模式

最后：不管选哪家，这三个维度一定要问清楚。欢迎评论区交流你们在用的系统。

## 五、配图完全指南

小红书是**封面驱动**的平台。首图不吸引，正文再好没人看。每一篇帖子至少需要 3-6 张图（首图 + 内页），图片质量直接决定点击率和涨粉速度。

---

### 5.1 首图类型与制作方法

按效果好→差排列，每种附具体做法：

#### 类型 1：真人+产品（效果最好，信任感最强）

**适合：** 故事帖、案例帖、观点帖
**怎么做：**
- 你（创始人）坐在电脑前，侧脸或半身，屏幕上是 UniMind 的界面
- iPhone 人像模式拍，自然光（窗户边），不要打灯
- 服装：smart casual（衬衫/素 T，不要西装——太像销售）
- 屏幕内容要清晰可辨：AI 出题中心 / 知识图谱 / Memorix 曲线
- 人物占画面 40-50%，屏幕占 30-40%，留白 10-20%

**首图上加的文字：**
- 2-3 行大字，放在人物/屏幕旁边的留白区域
- 字体：粗体无衬线，字号要大（占画面宽度 60-70%）
- 示例文字：**"做了一年 AI 教育 / 最让我意外的 / 不是技术是人心"**

**制作工具：** 拍好照片 → 打开醒图/黄油相机 → 加文字 → 加 UniMind 蓝色滤镜（#0F1729 透明度 15%）统一色调

#### 类型 2：对比图（点击率最高，信息差最强）

**适合：** 干货帖、种草帖、案例帖
**怎么做：**
- 左右分屏或上下分屏
- 左侧/上方：Before（痛点场景，用暗色调/灰色调）
- 右侧/下方：After（解决方案，用品牌蓝色调）
- 中间一个"→"或"VS"分隔

**示例布局：**
```
┌─────────────────────────┐
│   😰 手工出题            │
│   ┌───────────────┐     │
│   │  8小时/套      │     │
│   │  3个老师       │     │
│   │  60%准确率     │     │
│   │  无法个性化    │     │
│   └───────────────┘     │
│          ↓              │
│   🚀 AI 出题管线        │
│   ┌───────────────┐     │
│   │  30分钟审核    │     │
│   │  1个老师       │     │
│   │  85%+可用率    │     │
│   │  千人千面       │     │
│   └───────────────┘     │
└─────────────────────────┘
```

**制作工具：** Figma / Canva。用简单的色块 + 大号数字，不要花哨图标。数字是核心——大号加粗，颜色对比强烈（灰色 vs 蓝色）。

#### 类型 3：大字报（制作最快，适合干货帖）

**适合：** 干货帖、观点帖
**怎么做：**
- 纯色背景（深蓝 #0F1729 或浅灰 #F5F5F5）
- 一个核心观点，大字居中，占画面 60-70%
- 副标题小字在下方（15-20px）
- 底部 UniMind logo 小标
- 不要装饰元素，只用文字排版

**示例：
```
┌─────────────────────────┐
│                         │
│   你的机构老师          │
│   40% 的时间            │
│   没在教学              │
│                         │
│   在出题、批改、        │
│   做 Excel              │
│                         │
│   ──────────────────    │
│   UniMind               │
└─────────────────────────┘
```

**制作工具：** 醒图/黄油相机（手机 3 分钟搞定），或 Figma（更精细）。背景色固定为 #0F1729 或 #FFFFFF + 深蓝文字。

#### 类型 4：产品截图+场景包装（适合种草帖）

**适合：** 种草帖、功能展示帖
**怎么做：**
- 不要裸截图。给截图加一个使用场景的包装
- 方式 A：截图放在一个设备 mockup 里（MacBook/iPad 外壳）→ Figma 社区搜"device mockup"免费模板
- 方式 B：截图上方加一行场景描述文字（如"AI 3 分钟出了 50 道题，老师只需审核"）
- 方式 C：截图 + 手写风格标注（圈出关键功能，加箭头+文字说明）
- 产品界面截图不要缩小到看不清。保持原比例，确保文字可读

#### 类型 5：多图轮播（干货帖/教程帖的核心形式）

轮播图是小红书效果最好的内容形式之一。用户滑动 = 主动消费。

**轮播图结构（以 6 张为例）：**
| 图序 | 内容 | 设计要求 |
|------|------|----------|
| 图 1 | 首图（标题+钩子） | 大字，3 秒内看懂这篇讲什么 |
| 图 2 | 问题是什么 | 痛点描述，1 个数据或 1 个场景 |
| 图 3-4 | 为什么现状不行 | 每个原因一张图，不要挤 |
| 图 5-6 | 解决方案 | 具体做法 + 工具推荐（自然引出 UniMind） |
| 图 7（可选）| CTA | "你的机构呢？评论区聊聊" |

**轮播图设计统一性：**
- 所有图用同一底色（白底或浅灰底最安全）
- 每张图右上角小字标页码（1/6, 2/6…）—用户知道还有多少，更愿意滑完
- 每张图一个核心信息，字号保持一致

### 5.2 配色系统

```
品牌色板（严格统一，形成账号视觉记忆）：
━━━━━━━━━━━━━━━━━━━━━━━━
深蓝底   #0F1729  用于大字报背景、标题字
品牌蓝   #3B82F6  用于重点标注、数字高亮
浅蓝底   #EFF6FF  用于轮播图底色
白底     #FFFFFF  用于产品截图背景
灰色字   #6B7280  用于副标题、说明文字
深色字   #1F2937  用于正文
━━━━━━━━━━━━━━━━━━━━━━━━
```

**配色原则：**
- 80% 的图用白底/浅灰底 + 深蓝文字（干净、专业、耐看）
- 15% 的图用深蓝底 + 白字（强调、大字报、对比图的 After 侧）
- 5% 用品牌蓝点缀（数据高亮、重点圈注）
- **不要用渐变、不要用多种颜色混搭。** 两色足够。

### 5.3 字体规范

| 用途 | 字体 | 大小参考 |
|------|------|----------|
| 首图大标题 | 思源黑体 Heavy / 苹方粗体 | 占画面 50-70% 宽度 |
| 首图副标题 | 思源黑体 Regular | 主标题的 1/3 大小 |
| 轮播图标题 | 思源黑体 Bold | 画面顶部，占 20% 高度 |
| 轮播图正文 | 思源黑体 Regular | 16-20px（手机上可读） |
| 数据数字 | 思源黑体 Heavy + 品牌蓝 | 越大越好，数字是视觉锚点 |
| 标注/箭头 | 手写体（黄油相机"黄油溏心体"）| 小但可读 |

**字体原则：**
- 全文只用一种字体家族（思源黑体），通过粗细区分层级
- 不要用衬线体、不要用花体、不要用超过 2 种字体
- 手写标注体只在"标注截图"时用，不要用在标题

### 5.4 四类帖子的具体配图方案

#### 干货帖配图方案（轮播为主，6-9 张）

```
图 1：大字报首图（核心观点）
图 2：数据或事实（"一个老师 40% 时间不做教学"）
图 3：原因分析 1
图 4：原因分析 2
图 5：解决方案框架
图 6：具体做法/工具
图 7：CTA（引导评论）
```

**设计节奏：** 图 2-4 用浅色底（阅读感轻松），图 5-6 用白底（开始讲方案，视觉变干净），首图用大字报风格。

#### 种草帖配图方案（场景对比为主，4-6 张）

```
图 1：Before/After 对比首图
图 2：Before 场景细节（真实老师工作场景描述）
图 3：After 场景细节（用上 UniMind 之后）
图 4：产品截图+标注（AI 出题中心/判分界面/知识图谱）
图 5：数据对比（时间、准确率、留存率）
图 6：CTA
```

**设计重点：** Before 用灰色调，After 用品牌蓝白调。产品截图加手写标注（画圈+箭头+简短文字），显得真实在使用。

#### 故事帖配图方案（真人为主，3-5 张）

```
图 1：真人+产品照片（你在电脑前，屏幕亮着 UniMind）
图 2：故事相关图片（白板上的架构图、深夜代码截图、客户聊天记录打码）
图 3：关键数据或转折点（大字）
图 4：感悟/收获（文字为主）
```

**设计重点：** 真人照片不修图不滤镜（加一点亮度即可）。真实感 > 精致感。聊天记录截图打码客户信息后直接放，不要重新排版——真实聊天记录的说服力远超任何设计。

#### 案例帖配图方案（数据对比为主，3-5 张）

```
图 1：数据对比首图（核心数字大号："23%"）
图 2：客户场景描述（什么机构、什么科目、什么痛点）
图 3：用了什么功能（一一对应痛点和功能）
图 4：效果数据（Before/After 数字对比）
图 5：客户原话（打码头像+截图）
```

**设计重点：** 数字要大、要对比、要上色。23% 用品牌蓝，#F5F5F5 底色衬托。不要只写"效率提升"——写"出题时间从 8 小时降到 30 分钟"。

### 5.5 图片制作工具链

| 工具 | 用途 | 难度 | 推荐度 |
|------|------|------|--------|
| **醒图** | 首图加文字、滤镜、调色 | 零门槛，手机操作 | ⭐⭐⭐⭐⭐ 主力 |
| **黄油相机** | 手写标注、花字（标注截图用） | 零门槛，手机操作 | ⭐⭐⭐⭐⭐ 主力 |

| **Canva** | 快速套模板 | 零门槛 | ⭐⭐⭐ 应急用 |

**推荐工作流：**
1. CleanShot X 截产品界面 → 加边框/阴影
2. Figma 做好 3-5 套轮播图模板（干货模板、种草模板、故事模板、案例模板各一套）
3. 每次发帖：打开 Figma 模板 → 改文字和截图 → 导出 PNG
4. 首图如需真人照片：iPhone 拍 → 醒图加文字 → 导出
5. 这个流程跑通后，做一套图 15-30 分钟

### 5.6 不要做的事

- **不要用 stock photo。** 培训机构老板一眼能认出"这不是真人"。你用真实照片可能不够美，但信任感完胜。
- **不要用 AI 生成的图。** 小红书用户对 AI 图极度敏感且反感。特别是 AI 生成的人脸——会被评论区群嘲。
- **不要文字堆满。** 一张图只有一个核心信息。想讲 3 个点就做 3 张图，不要挤在一张里。
- **不要多种配色混搭。** 一个账号一个色板。今天红明天蓝 = 没有品牌记忆。
- **不要用复杂图表。** 培训机构老板不是数据分析师。数字要大、对比要直观、一图一事。
- **不要裸发产品截图不加包装。** 纯功能界面 = 广告 = 划走。截图必须配场景、配标注、配对比。

## 九、转化漏斗

```
小红书帖子（持续发）
    ↓ 种草 / 信任建立
私信咨询（或评论"怎么试用"）
    ↓
转到微信 / Demo 预约
    ↓
桌面 App Demo（15-20 分钟）
    ↓
免费试用期（用机构的真实学科内容跑一遍）
    ↓
付费转化
```

**关键节点：**
- **帖子 → 私信**：每条帖子末尾要有"钩子"——"评论/私信XXX我发你"
- **私信 → Demo**：不要在私信里推销。先问对方机构情况，再针对性介绍。用 `docs/X出海商业化打法.md` 的 DM 策略
- **Demo → 试用**：Demo 结束当场给邀请码。别拖
- **试用 → 付费**：试用期间关注使用数据，主动问反馈。不用的客户要问为什么

### 私信回复模板

**对方问"怎么试用"时：**
> 你好！先了解一下你们的情况——是什么类型的机构？主要教什么科目？大概多少学生？我看看适不适合你们用 😊

**对方回复后（如果是目标客户）：**
> 了解了。你们这个场景我们刚好能帮上。[具体说 1-2 个能解决的痛点]。要不约个 15 分钟线上 demo？我用你们的学科内容实际跑一遍给你看。你觉得有用再聊后续，没用也不浪费你时间。

**对方犹豫时：**
> 没事，先不急。我刚发了篇关于[相关痛点]的文章，你看看有没有参考价值。[发链接]。什么时候想聊随时找我。

## 十一、不要做的事

- **不要一上来就卖**。小红书用户对硬广极度敏感。前 2 周专注提供价值，建立"这个人懂行"的认知。
- **不要只发产品截图**。裸产品截图 = 广告。产品截图必须配合使用场景和真实感受。
- **不要追求精致**。小红书的调性是"精致的真实"，不是"广告公司的精致"。iPhone 拍的实况照片 > 专业相机精修图。
- **不要忽略评论区**。评论区的互动权重极高。每一条评论都回复——这比多发一篇帖子的效果还好。
- **不要搬运 X 的内容直接翻译过来**。两个平台的语境、语言、节奏完全不同。
- **不要在帖子里放微信号**。小红书对导流敏感。用"私信我""评论区告诉我"代替。
- **不要停更超过一周**。小红书算法对更新频率敏感。如果忙，一周至少发 2 篇保持账号活跃。
