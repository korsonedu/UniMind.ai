# 知识图谱个性化标注与自适应推荐 (Personalized Knowledge Graph)

## 1. 背景与动机
目前系统的知识图谱主要用于静态展示考研 431 的大纲结构（如“宏观经济学 -> 货币需求理论 -> 鲍莫尔-托宾模型”）。
为了实现真正的“因材施教”，我们需要将图谱从**“静态知识字典”**升级为**“动态学习雷达”**。本设计将用户的 FSRS 数据、错题数据映射到图谱节点上，实现可视化的掌握度热力图和智能学习路径推荐。

## 2. 核心设计：动态状态映射 (Dynamic State Mapping)

每个图谱节点 (Node) 对应一个 `KnowledgePoint`。我们需要计算该节点针对当前用户的**综合掌握度评分 (Mastery Score)**。

### 2.1 掌握度计算模型
掌握度 $M$ 综合考量以下指标：
1. **FSRS 召回率 (Retrievability, $R$)**：反映该节点下记忆类知识的遗忘程度。
2. **答题胜率 (Win Rate, $W$)**：该节点下关联题目（尤其是计算题/综合题）的历史正确率。
3. **ELO 积分 (Elo Rating, $E$)**：反映用户在该难度层级下的竞技水平。

算法简化示例：
$$ Mastery = \alpha \times \bar{R} + \beta \times W + \gamma \times \text{Normalize}(E) $$
（$\alpha, \beta, \gamma$ 为权重参数，初始可设定为 0.4, 0.4, 0.2）

### 2.2 图谱热力图渲染 (Heatmap Visualization)
在前端（如使用 ECharts, G6 或 D3.js）渲染图谱时，根据 Mastery 赋予节点不同的视觉反馈：
- $M \ge 0.85$：**精通 (绿色)** —— 节点发光或标记为“已攻克”。
- $0.6 \le M < 0.85$：**复习 (黄色)** —— 需要定期的 FSRS 复习。
- $M < 0.6$：**薄弱 (红色)** —— 核心丢分点。

## 3. AI 驱动的自适应路径规划 (Adaptive Pathing)

### 3.1 阻塞节点识别 (Bottleneck Detection)
系统遍历有向无环图 (DAG) 结构。如果用户在高级节点（如“资本资产定价模型 MM 理论”）反复做错，算法会回溯其前置节点（如“套利定价”、“公司税”）。
如果发现前置节点处于“红色”状态，系统会判定该前置节点为**“阻塞节点 (Bottleneck)”**。

### 3.2 学习流干预
- **提示**：“检测到您在 [MM理论] 频繁出错，我们发现您的前置知识 [无税假定] 尚未掌握，建议先跳转至该节点复习。”
- **动态推题**：AI 会暂时暂停推送 MM 理论的难题，转而推送前置知识的练习题。

## 4. 用户共创与个性化关联 (User Annotations)

允许用户在知识节点上进行以下操作：
1. **挂载私有笔记 (Personal Notes)**：在节点上绑定自己的助记口诀、Markdown 笔记。
2. **建立自定义边 (Custom Edges)**：用户发现节点 A 和节点 B 之间有自己理解的独特联系，可以手动连线（例如：“货币政策” 与 “汇率决定理论”的传导机制）。
3. **社区精选标注**：优秀的个人笔记经审核后，可升级为“社区高赞标注”，供其他同学参考。

## 5. 落地步骤 (Implementation Steps)

1. **数据库层**：在 `UserKnowledgeState` 表中新增 `mastery_score` 字段，通过 Celery 定时从 FSRS 聚合计算。
2. **API 层**：提供 `/api/graph/user_heatmap/` 接口，返回包含颜色/权重状态的图谱 JSON。
3. **前端层**：改造图谱组件，支持颜色渲染、点击节点查看个人笔记。
4. **AI 层**：实现基于 DAG 的前置节点回溯算法。
