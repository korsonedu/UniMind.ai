# 主观题 AI 阅卷引擎 (AI Essay Grading Engine)

## 1. 背景与动机
431 金融学综合考试中，名词解释、简答题和论述题占据了极高的分值比例。目前市面上的刷题软件大多只能处理客观题（单选、多选、判断）。学生在练习主观题时，往往只能“看参考答案自己估分”，缺乏客观的评价和反馈机制。
本系统旨在引入 AI 阅卷引擎，通过大模型强大的语义理解能力，对学生的主观文本进行结构化批改。

## 2. 核心架构设计

### 2.1 采分点模型 (Rubric-based Scoring)
主观题批改不能是黑盒，必须基于“采分点”。
在数据库 `Question` 模型中，针对主观题，新增一个 `rubric` (JSON) 字段，存储官方给出的按点得分标准：
```json
[
  {"point": "提到无税假设", "score": 2, "keywords": ["无税", "不考虑税收"]},
  {"point": "说明资本结构与企业价值无关", "score": 3, "keywords": ["无关", "不影响", "饼图理论"]},
  {"point": "得出公式或推导", "score": 2, "keywords": ["Vl = Vu"]}
]
```

### 2.2 阅卷管线 (Grading Pipeline)
当学生提交主观题答案后，触发阅卷管线：
1. **语义匹配**：不要求字面完全一致。使用 LLM 对照 `rubric` 中的每一个点，评估学生的答案是否在语义上命中了该得分点。
2. **判分**：根据命中的采分点，累加得出总分。
3. **批注生成 (Annotations)**：LLM 返回一个批注数组，指出学生答案中缺失了哪些点，或者哪一部分逻辑存在冗余。

## 3. API 与数据流

- **输入**：`{ "question_id": 102, "user_answer": "MM理论认为企业价值和怎么融资没关系..." }`
- **大模型 Prompt 模板 (`ESSAY_GRADER`)**：
  > 你是一位严谨的金融学考研阅卷老师。
  > 题目：{{ question_content }}
  > 采分点标准：{{ rubric_json }}
  > 考生答案：{{ user_answer }}
  > 请严格按照采分点进行打分，不要因为考生写得多就给同情分。输出每项得分情况、总分以及具体的改进建议。
- **输出结构**：
  ```json
  {
    "total_score": 5,
    "max_score": 7,
    "details": [
      {"point": "提到无税假设", "earned": 0, "reason": "考生未提及是否含税的前提条件"},
      {"point": "说明资本结构与企业价值无关", "earned": 3, "reason": "考生明确指出了两者无关"},
      {"point": "得出公式或推导", "earned": 2, "reason": "虽然未写标准公式，但语义表达了企业总价值固定的意思"}
    ],
    "overall_feedback": "核心概念理解正确，但在学术表达上不够严谨，必须明确前置假设（无税/有税）。"
  }
  ```

## 4. 前端展示
- 在答题结果页，左侧展示学生的原始答案，右侧展示打分明细。
- **高亮标红**：将未得分的采分点在参考答案中高亮显示，强迫学生关注遗漏的知识区块。
