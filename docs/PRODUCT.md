## 一、UniMind 是什么

UniMind 是一套面向教育培训机构的 AI 教学基础设施。不是题库，不是 LMS，不是网校——是三者之上的**智能调度层**。

核心能力：**Memorix 自进化记忆调度算法**。传统教学最大的浪费不是课程差，是学生忘了。遗忘不可怕，可怕的是你不知道他什么时候忘。Memorix 基于每个学生每道题的实时数据，动态计算最佳复习时机——在遗忘临界点精准切入。这不是「今天该复习了」的粗粒度提醒，是每道题有自己独立的最优间隔。

## 三、双 Agent 体系

### 小宇 · 学生端 AI 学习教练

学生的专属陪练。自主调用 17 个工具：抽题、评分、讲解、知识图谱渲染、学习数据分析。

- 主动推送到期复习题，不是被动等学生来
- 错因分析（概念错误/计算失误/粗心），不同原因不同策略
- 对话中嵌入可视化卡片、题目面板、知识图谱
- 自适应对话风格：公式推导型 / 样例驱动型 / 鼓励型 / 直接型

### 命题官 · 教师端教研助手

教师的出题和教学管理指挥中心。

- **快速出题**：指定知识点+题型+难度，秒级生成
- **ARC 精修**：4-Agent 对抗管线，Author→Reviewer→Revise→Classify，85%+ 通过率
- **班级分析**：薄弱点分布、正确率对比、个体差异定位
- **维护中心**：资产管理、知识树编辑、通知配置

## 四、算法层（机构教学效果的护城河）

### Memorix 记忆调度

- **图扩散（Memorix-Field）**：知识图谱结构信息融入调度。如果学生在「导数」表现好，相关知识点「微分」的复习优先级自动调整
- **再巩固窗口（REC）**：在记忆不稳定期精确切入，而非固定间隔
- **错因感知**：概念错误缩短间隔重新强化，计算错误加同类变式题

### IRT 认知诊断

- 3PL 三参数 Logistic 模型：区分度 / 难度 / 猜测概率
- **机构级隔离**：同一道题在不同机构（重点校 vs 普通校）有独立的参数估计
- θ 能力值按知识点粒度输出，支持 kp_breakdown

### 变式题匹配

- 学生做错 → 自动匹配同知识点、相近难度的变式题
- 插入下次复习队列，避免「记住答案测不出真实掌握」

### GEPA 自进化

- Trajectory 数据收集 → 周度分析 → 自动生成参数优化建议
- 人工审核后应用，不自动变更

## 八、技术保障

- 生产环境：阿里云 ECS + PostgreSQL 16 + Redis + Nginx
- 熔断保护：AI 调用按任务粒度隔离，单模型故障不扩散
- 多租户隔离：机构数据在 ORM 层强制过滤，零跨机构泄露路径
- Cookie 认证：httpOnly cookie + CSRF 保护，前端不存 token
- 敏感字段加密：Fernet AES 对称加密

## 十、常见问题

**Q: 机构可以用自己的教材和知识树吗？**  
A: 可以。知识树编辑器支持自定义学科体系。题目入库必须关联知识点——不同机构的知识体系完全独立。

**Q: 学生的数据安全吗？**  
A: 机构间数据在数据库层强制隔离。学生答题数据仅本机构教师可见。

**Q: 小宇能教多深的内容？**  
A: K12 + 大学基础学科全覆盖。题目由你配置的知识点和难度决定。

**Q: 如何量化学生进步？**  
A: 三个维度：Memorix stability 增长斜率（记忆是否更稳定）、IRT θ 值变化（能力是否提升）、知识图谱掌握度热力图（薄弱点是否收窄）。
