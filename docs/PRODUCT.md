# UniMind — Agent 驱动的新一代智能教育基础设施

UniMind 是一套面向教育培训机构的 AI 基础设施。系统部署两个自治 AI Agent——学生学习教练「小宇」与教师教研助手「工作台」——覆盖从出题、刷题、评分批改到知识追踪的全链路。底层为四层自适应学习架构，支撑 12 学科 11,908 知识点的全景知识图谱。B2B 机构部署，数据完全隔离。

---

## 一、UniMind 是什么：产品哲学与系统定位

### 不是菜单，是对话

UniMind 区别于常见的「菜单驱动 SaaS + AI 聊天框」架构。在那种架构中，用户登录后面对仪表盘、侧边栏和层层菜单，从题库、作业、班级管理等功能模块中自行导航，AI 聊天框是其中一个附加功能。

UniMind 的架构逻辑相反：**Agent 是入口，SaaS 是画布。** 学生登录后首先面对的是小宇——小宇根据记忆模型和薄弱点，推送该做的题、提示该复习的知识点、展示掌握度热力图。学生不需要知道"题库""错题本""知识图谱"这些功能的名字，小宇在对话中调出对应的卡片和面板。教师同理——工作台是对话式指挥中心，"出 5 道极限计算题"一句话触发完整的出题管线。

Agent 为入口、SaaS 为画布——这不是交互偏好，是产品架构的根本选择。

### 两种架构逻辑

> 传统：功能模块 → 用户自学菜单 → 手动操作
> UniMind：Agent 理解意图 → 编排工具 → 对话中交付结果

这带来两个根本差异。

**第一，系统能力的边界不取决于功能菜单的数量。** 传统 SaaS 每增加一个能力，需要新页面、新按钮、新引导流程。UniMind 每增加一个能力，只需给 Agent 注册一个工具——学生在对话中说出需求，Agent 调用对应工具。能力的增长体现在 Agent 工具箱的扩展，而非菜单数量的膨胀。

**第二，用户不需要自行决策。** 在传统 SaaS 中，学生需要判断"我该去错题本看看"还是"该做练习题"，这个判断需要学习能力和自驱力。在 UniMind 中，小宇基于记忆模型替学生做了这个判断——"三角函数你有 3 道到期题，上次导数正确率从 40% 到 65%，今天继续？"学生回应一句话即可。

**UniMind 本质上是一个「有结构化能力的 AI 学习伙伴」——对话是前台，SaaS 是后台。** 题库、知识图谱、算法引擎、机构管理是后台基础设施，用户直接面对的是小宇和工作台。

## 二、机构使用 UniMind 能获得什么

### 对学生的价值

**不再遗忘**。传统刷题模式下，学生做完一轮题后何时复习全凭自觉。UniMind 的 Memorix 引擎为每道题独立计算遗忘曲线，在记忆临界点自动推送复习。学生打开小宇时面对的永远是当前最优的练习组合。

**精准诊断**。错因自动归类为概念错误、计算失误或粗心，三种错因触发不同的再巩固策略。IRT 模型给出机构级标准化的能力值，而非粗糙的正确率百分比。

**主动推动**。即使学生不登录，调度执行器在后台轮询到期题目数，超阈值时自动推送飞书或邮件通知。学生不需要自驱力来维持学习频率——系统替他记住该做什么。

### 对教师的价值

**出题效率提升**。传统模式下教师手工出题或从题库筛选，一套 20 题的练习卷耗时 1-2 小时。工作台一句话出题，秒级生成，ARC 精修管线保证 85% 以上的一次通过率。

**学情可视化**。班级薄弱知识点分布、学生 × 作业成绩矩阵、个体学习趋势，全部通过自然语言查询即时呈现。教师不需要学会操作任何数据平台。

**批改减负**。客观题自动批改，主观题 AI 预评分后教师复核。一个 30 人班级的作业批改从 2 小时降到 15 分钟。

### 对机构的价值

**续费率提升**。学生因为遗忘导致挫败感是退课的首要原因。Memorix 驱动的主动复习机制维持学习效果的正向感知，直接提升续费率。

**教师产能释放**。教师从出题和批改中释放的时间可用于答疑和个性化辅导，提升机构的服务深度和差异化竞争力。

**数据资产沉淀**。每个学生的记忆模型、IRT 能力值、知识点掌握图谱随使用持续积累。机构离任教师不会带走这些数据——教学资产留在系统里。

## 三、典型使用场景

### 学生的一天

小宇根据记忆模型检测到 5 道三角函数到期题，在后台触发飞书推送。学生打开 UniMind，小宇已经在对话中："三角函数 5 道到期题，上次正确率 60%，现在复习？"学生回复"好"，小宇逐题推送。做到第 3 题出错——小宇立即归类为概念错误，展示解析后从变式题库匹配一道相近难度的题插入队列。做完后小宇展示今日学习摘要和掌握度变化。全程不需要点击任何菜单。

### 教师的一天

教师打开工作台说"出 3 道极限计算题，中等难度，给火箭班做练习"。工作台搜索知识点、生成题目，在对话中展示题目预览卡片。教师确认后说"入库，发给火箭班"，工作台创建作业、绑定班级、发布，返回"已发布 3 题给火箭班，23 名学生"。第二天教师说"火箭班作业交了没"，工作台返回提交进度和待批改列表。教师逐个批改，AI 辅助评分。

### 新机构接入

1. 注册机构账号 → 选择学科模板（或导入自定义知识树）→ 30 秒完成初始化
2. 创建班级，导入学生名单，邀请教师加入
3. 教师通过对话出题或导入已有题库
4. 学生通过邀请链接注册，完成诊断测试
5. 小宇即刻开始主动推送复习任务

从注册到学生开始使用，全程不超过一个工作日。14 天全功能试用期间，机构可评估效果后决定付费方案。

## 四、系统架构

UniMind 的四层架构自底向上逐层支撑。每一层有独立的学科基础和设计原则。

**第一层 · 基础框架** — 通用化，与教育业务无关。

这一层定义了 UniMind 作为 Agent 系统的最底层能力：对话生命周期管理、Bot 注册与运行时、工具注册与权限控制、意图预筛选路由、模型分级调度、WebSocket 与 SSE 传输层。这些是任何 Agent 驱动系统都需要的基础设施，不涉及教学逻辑。

框架层的核心设计原则是解耦——新增一个 Agent 只需注册 BotProfile 与工具集，自动继承上层全部能力，不需要重建对话系统或 API。调度执行器在后台轮询到期题目数，超阈值时主动推送通知拉回学生。智能工具路由基于 SkillRouter 论文实现：Embedding 检索 + 关键词 Fallback 将 40 项工具缩减为 5-8 个场景候选，本地 BGE 模型在 Django 启动时预加载。

**第二层 · 记忆系统** — 系统的有效性取决于这一层。

这一层解决的核心问题是：系统对学生知道多少。传统系统只记录"做对了几道题"，UniMind 为每位学生每道题独立维护一个持续演化的记忆模型——stability（记忆稳定性）、difficulty（题目难度感知）、next_review_at（下次复习时机），基于 Weibull 遗忘模型的在线 SGD 参数更新。Memorix-Field 图扩散变体将知识图谱结构信息融入调度决策，跨知识点传播难度估计。再巩固窗口（REC）在记忆不稳定期精确切入复习。三种错因（概念错误、计算失误、粗心）触发不同的再巩固策略。

MemorySystem 查询接口层向上暴露统一的数据访问契约，所有上层能力通过同一接口获取学生状态。自适应 Prompt 系统在每次对话前构建双层上下文注入 system prompt——同一个学生两次打开小宇，小宇的话术和策略已经在变化。

MUTAR 自进化架构位于这一层。Trajectory 数据自动采集 → 周期性分析（工具调用成功率、对话完成度、prompt_variant 对比）→ 自动生成优化候选（Memorix 参数调整、Prompt 改进、Bot 审查）→ 存储至 Redis 供人工审核。不自动应用，人始终在回路中。记忆系统不是静态存储——它随使用持续进化。

**第三层 · 教育能力闭环** — 有效教育理论的工程化实现。

这一层承载完整的教学逻辑，底层是成熟的教育理论框架。教（知识讲解、资源推荐）、练（抽题练习、到期复习）、测（诊断测试、模拟考试）、评（自动批改、错因分析）——四个环节构成教学闭环。这不是拍脑袋的功能堆砌，而是基于掌握学习理论、间隔重复原理和形成性评估框架的工程化实现。

闭环的意义在于积累效应：评测结果回流至记忆系统更新模型参数，模型的变化影响下一次练习的题目选择和复习时机。学生使用越久，系统对学生的建模越准，教学干预越精确。4-Agent ARC 出题管线（Author → Reviewer → AuthorRevise → Classifier，最多 3 轮迭代）位于这一层，通过对抗博弈将单次出题通过率从 60% 提升至 85% 以上。

**第四层 · IRT 认知诊断** — 心理测量学的标准化框架。

这一层解决的核心问题是：如何科学地衡量学生的能力，以及题目的质量。UniMind 采用项目反应理论（IRT，Item Response Theory）的三参数 Logistic 模型——这是心理测量学领域经过 80 年验证的标准化框架，广泛应用于 TOEFL、GRE、PISA 等大规模标准化考试。

IRT 的核心优势在于参数不变性：学生的能力估计不依赖于具体做了哪道题，题目的难度估计不依赖于被哪些学生做过。UniMind 在此基础上实现了机构级参数隔离——同一道题在重点中学与普通中学拥有独立的区分度、难度和猜测参数。采用 MML-EM 参数估计方法，输出 θ 能力值与知识点粒度掌握度分解。Fallback CTT 保证新题冷启动。

变式题匹配引擎位于这一层：错题后自动检索同知识点相近难度题目插入下次复习队列，基于 IRT 难度参数进行匹配而非随机选取。

核心原则：**Memorix 是闭环的节拍器。** 系统不做立刻纠错——错题按间隔重复节奏自然出现。IRT 负责诊断和测量，Memorix 负责调度时机。

## 六、数据安全与合规

**物理隔离**：每家机构拥有独立的数据空间，包括学生记忆模型、IRT 参数、知识图谱定制与 Agent 对话历史。机构间数据不可互相访问。

**私有化部署**：Pro 方案支持完整私有化部署，数据不出机构内网。提供 systemd 配置、Nginx 反向代理方案与 PostgreSQL 初始化脚本。

**数据主权**：机构拥有全部数据的所有权和导出权。支持按学生、按班级、按时间范围导出学习数据、考试成绩和对话记录。

**模型安全**：AI 引擎设置按任务类型粒度的熔断器，单模型故障不扩散。所有 AI 调用经过工具权限白名单审计，Agent 无法越权操作。

## 十、常见问题

**Q: 现有题库可以导入吗？**
A: 支持 Markdown 格式批量导入。知识树和题目均可按学科独立管理，机构可混合使用系统预置和自有内容。

**Q: 数据存储在哪里？如何保证安全？**
A: SaaS 模式数据存储于机构专属 PostgreSQL 数据库，机构间物理隔离。Pro 方案支持完整私有化部署。

**Q: 教师需要培训吗？**
A: 不需要。工作台是对话式操作，教师像和人说话一样布置任务。新教师通常在首次使用后 5 分钟内掌握基本操作。

**Q: 学生的基础水平参差不齐怎么办？**
A: 新学生首次登录必须完成诊断测试，系统根据测试结果初始化个人记忆模型。后续所有训练和复习均基于个人模型调参，不同水平的学生在同一班级中获得不同的练习组合。

**Q: 支持哪些平台？**
A: Web 端全功能支持，移动端适配中。飞书和邮件推送已上线。

**Q: 是否支持更换 AI 模型？**
A: 系统支持供应商热插拔，可切换 DeepSeek、OpenAI 等兼容接口的模型。多模型分级路由允许不同任务使用不同模型。
