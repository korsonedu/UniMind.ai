# 小宇 Copilot — 全局伴随式 AI 助手

**Date:** 2026-06-10
**Status:** Draft
**Scope:** 全局组件 + 刷题/课程页上下文注入 + 侧边栏 UI

## 背景

小宇当前仅存在于 `/xiaoyu` 页面，用户需要主动切换才能使用。产品问题是**粘性不足** — 用户用完即走，未形成习惯。

核心转变：小宇从"目的地型 Agent"变为"伴随型 Copilot" — 始终在侧，随时可达，上下文感知。

## 设计决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 产品定位 | 全局伴随 Copilot | 降低访问摩擦，提升粘性 |
| `/xiaoyu` 关系 | 保留，Copilot 是快捷入口 | 深度操作（学习计划、可视化）仍在主页面 |
| 对话历史 | 共享 | Copilot 和 `/xiaoyu` 是同一个 Agent 的不同 UI |
| 唤起方式 | 悬浮球 + 键盘快捷键 | 覆盖鼠标/键盘两类用户习惯 |
| 交互形态 | 右侧侧边栏 | 不遮挡主内容，可边看边问 |
| 显示范围 | 全站常驻（登录后） | 用户形成肌肉记忆："有问题找右下角" |
| 侧边栏宽度 | 固定 400px | 简单可靠，避免 resize 复杂度 |
| 页面切换 | 侧边栏保持打开，对话继续 | 不中断用户思路 |
| 关闭再打开 | 恢复对话 | 对话历史共享，保持连续性 |
| MVP 工具范围 | 裁剪为答疑工具 | 侧边栏空间有限，重操作引导回 `/xiaoyu` |

## 上下文传递

| 页面 | 自动传递的上下文 | 数据来源 |
|------|-----------------|---------|
| 刷题页 `/quizzes` | 题目内容 + 用户已选答案 + 正确答案 | 前端 state |
| 课程/视频页 `/courses/:id` | 课程ID + 当前播放时间点 | 播放器 state |
| 其他页面 | 无特殊上下文 | — |

不传最小（只传ID），因为会多一轮工具调用增加延迟。前端已有数据直接传。

## MVP 工具范围（Phase 1）

侧边栏保留的工具：
- `explain_question` — 讲解题目
- `search_questions` — 搜索相关题目
- `query_knowledge_tree` — 查询知识点

重操作引导回 `/xiaoyu`：
- 生成可视化 → "去小宇主页面查看完整可视化"
- 管理学习计划 → "去小宇主页面管理计划"
- 数据分析 → "去小宇主页面查看分析报告"

## 技术方案

### 前端

**全局组件** `XiaoyuCopilot`，挂载在 `App.tsx` 最外层（登录后）：

```
App.tsx
├── <BrowserRouter>
│   ├── <Routes>...</Routes>
│   ├── <XiaoyuCopilot />    ← 全局挂载
│   └── ...
```

**组件结构**：
- `XiaoyuCopilot.tsx` — 主容器，管理 open/close 状态
- `CopilotFloatingBall.tsx` — 悬浮球，右下角固定定位
- `CopilotSidePanel.tsx` — 侧边栏，右侧滑出
- `CopilotChat.tsx` — 对话区，复用现有 chat 组件逻辑

**路由上下文检测**：
- 使用 `useLocation()` 检测当前路由
- 刷题页：从 quiz state 读取题目+答案
- 课程页：从 video player ref 读取播放时间
- 其他页：不传额外上下文

**API 复用**：
- 调用现有 `/api/ai/chat/stream/` (SSE)
- bot_id 固定为小宇的 bot_id
- 上下文通过 message 的 metadata 字段传递

**快捷键**：
- `Cmd+K`（Mac）/ `Ctrl+K`（Windows）唤起/关闭
- 使用 `useEffect` + `keydown` 事件监听

### 后端

**Phase 1 无需后端改动**：
- 复用现有 chat API
- 上下文通过前端 metadata 传递，后端已支持
- 对话历史共享机制已存在

**Phase 2 可选优化**：
- 后端根据 metadata 中的 page_type 自动筛选可用工具子集
- 上下文敏感的 system prompt 注入

## 分阶段实施

### Phase 1 — 刷题页试点（1-2天）

**目标**：验证"降低访问摩擦是否提升粘性"

**范围**：
- [ ] `XiaoyuCopilot` 全局组件框架
- [ ] 悬浮球 + 侧边栏 UI
- [ ] 刷题页上下文传递（题目+答案）
- [ ] 工具裁剪（仅答疑相关）
- [ ] `Cmd+K` 快捷键
- [ ] 暂不传课程页上下文

**验证标准**：
- [ ] 悬浮球在刷题页显示，点击展开侧边栏
- [ ] 侧边栏自动获取当前题目上下文
- [ ] 小宇回答时引用当前题目内容
- [ ] 重操作提示引导回 `/xiaoyu`
- [ ] `Cmd+K` 可唤起/关闭
- [ ] 页面切换侧边栏保持打开
- [ ] `make frontend-check` 通过

### Phase 2 — 全站铺开（3-5天）

**Phase 1 验证有效后实施**

**范围**：
- [ ] 课程页上下文传递（视频ID+时间点）
- [ ] 全站悬浮球（登录后所有页面）
- [ ] 非学习页基础问答（无特殊上下文）
- [ ] 侧边栏状态持久化（localStorage）
- [ ] 移动端适配（侧边栏全屏）

## 成功指标

| 指标 | 定义 | 目标 |
|------|------|------|
| Copilot 使用率 | 刷题页用户中，唤起 Copilot 的比例 | > 20% |
| 日均对话轮数 | Copilot 场景下的平均对话轮数 | > 3 轮 |
| 回访率 | 7天内再次使用 Copilot 的用户比例 | > 30% |
| `/xiaoyu` 导流 | 从 Copilot 引导到 `/xiaoyu` 的点击率 | 观察 |

## 不变项

- `/xiaoyu` 页面完整功能
- 小宇的 17 个工具能力
- 对话历史存储机制
- WebSocket / SSE 连接方式
- Bot 注册表结构
