## 灵感列表

### #001 分步推导可交互
**日期**：2026-05-30
**一句话**：小宇 Dashboard 中的分步推导可以点击任意一步，展开更细的讲解。
**为什么**：当前分步推导是一次性输出，用户看不懂某一步时只能重新提问。如果每一步可交互，用户能按需深入，学习效率更高。
**设想**：
- Visual Canvas 的推导步骤支持点击展开
- 点击某一步后，小宇针对该步生成更详细的子推导（可能包含图示、类比、前置知识回顾）
- 支持"追问"按钮，直接以该步为上下文继续对话
- 技术上：每步带 `step_id`，前端发 `/api/ai/chat/stream/` 时附带 `context_step_id`

**状态**：💡 待论证

### #002 错题 AI 复盘教练
**日期**：2026-05-30
**一句话**：刷完一套题后，AI 主动分析错题模式，生成个性化复盘报告 + 针对性练习。
**为什么**：学生刷题后通常只是看解析，缺乏对"为什么错"的深层反思。错题复盘是从 60 分到 90 分的关键环节，但很少有产品做得好。
**设想**：
- 做题完成后自动触发（或用户手动触发）AI 复盘
- 分析维度：错题知识点聚类、错误类型（概念混淆 / 计算失误 / 审题不清）、与历史错题的关联
- 输出：一段教练式对话（"我发现你连续三次在 XX 知识点犯了同类错误..."）+ 推荐 3-5 道针对性变式题
- 技术上：复用 Memorix 的遗忘曲线数据 + 知识树拓扑 + 出题管线

**状态**：💡 待论证

### #003 语音对话学习
**日期**：2026-05-30
**一句话**：小宇支持语音输入+语音回答，变成一个随时在线的AI家教，通勤/走路时也能学。
**为什么**：大量学习场景是碎片化的（通勤、排队、睡前），但当前只能打字交互。语音降低使用门槛，让学生每天多出 30-60 分钟的有效学习时间。且语音对话天然适合"讲题"场景——老师讲题本来就是用嘴说的。
**设想**：
- 前端：长按录音 → ASR 转文字 → 发给小宇 → TTS 朗读回答
- 支持打断（用户说"等等，上一步没听懂"，AI 回溯）
- 语音模式下自动切换为口语化回答风格（少公式、多类比）
- 可选：接入实时语音 API（如 DeepSeek Audio / 豆包语音），实现低延迟双向对话
- 技术上：前端 WebRTC 录音 + 后端 ASR/TTS（已有 ASR 基础设施 in courses）+ 流式 SSE

**状态**：💡 待论证

### #004 学习周报 / 月报
**日期**：2026-05-30
**一句话**：每周/每月自动生成个性化学习报告，邮件推送给学生和家长，像健身 App 的数据周报。
**为什么**：学生缺乏对自身学习状态的量化感知。"我这周学了多少""比上周进步了吗""哪里还在原地踏步"——这些问题当前无法直观回答。周报能制造"回顾仪式感"，提升留存；家长收到报告也能增加信任感和续费意愿。
**设想**：
- 数据源：做题记录、学习时长、知识点掌握度变化（Memorix）、AI 对话频次
- 报告内容：本周学习时长 vs 上周、正确率趋势、Top 3 进步知识点、Top 3 薄弱知识点、小宇的一句话评语
- 推送：邮件（已有 Resend 邮件基础设施）+ 站内通知
- 家长视角：绑定家长邮箱后自动同步报告
- 技术上：Celery 定时任务 + 已有 analytics 数据聚合 + 邮件模板

**状态**：💡 待论证

### #005 AI 出题市场
**日期**：2026-05-30
**一句话**：老师可以用命题官出题后，将优质题目上架到平台市场，其他机构付费使用。
**为什么**：命题官的出题质量已经很高，但优质题目的价值被锁在单个机构内。如果开放市场，好老师能变现，平台能形成内容飞轮，小机构能低成本获取高质量题库。
**设想**：
- 命题官出题后，老师可一键"上架"（设置价格 / 免费 / 积分兑换）
- 题目经过 ARC 管线审核 + 平台人工抽检后进入市场
- 购买方可以预览、试做、评分
- 平台抽成 20-30%，老师获得收益
- 技术上：已有出题 + 模板系统，新增市场模块（订单 + 审核 + 分成）

**状态**：💡 待论证
