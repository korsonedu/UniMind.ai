# 小宇前端形态设计

> 状态：Design | 日期：2026-06-06
> 对应架构：xiaoyu-four-layer-architecture.md

## 核心决定

**对话流 + 全屏专注模式混合。** 日常互动在对话流中，做题时自然过渡到全屏。卡片是闭环各阶段的 UI 锚点。

---

## 页面结构

```
/xiaoyu                          对话流（主界面）
/xiaoyu/practice/:sessionId      全屏做题页（独立路由）
      ↓ 提交后
/xiaoyu?practiceResult=:sessionId  对话流自动聚焦批改结果
```

---

## 对话流（主界面）

单栏布局，小宇消息 + 内联卡片 + 输入框。

### 卡片系统

四种卡片类型，对应教育闭环的不同阶段：

| 卡片 | 触发时机 | 内容 | 可操作 |
|------|---------|------|--------|
| **练习卡片** | 小宇推荐练习 | 题目数量 + 知识点 + 预计时间 + 「开始练习」按钮 | 点击 → 全屏做题 |
| **结果卡片** | 提交答题后 | 正确率 + 得分 + 用时 + 「查看分析」按钮 | 点击 → 小宇分析 |
| **纠错卡片** | 小宇分析错因后 | 针对性题目预览 + 「开始纠错」按钮 | 点击 → 全屏做题 |
| **数据卡片** | 小宇分析数据时 | 统计/图谱/趋势（可视化渲染） | 静态展示 |

卡片内联嵌入对话流，由小宇通过 `render_visual` (action_cards / data_card) 生成。

### 对话流状态

- **首次进入**：小宇主动开口，基于记忆的个性化问候 + 数据洞察
- **有进行中练习**：显示上次未完成练习的状态
- **刚完成练习**：URL 参数 `?practiceResult=:sessionId` → 自动展示结果卡片 + 小宇分析
- **空闲状态**：小宇等待用户提问，8 个技能胶囊可见

## 全屏做题页

独立路由 `/xiaoyu/practice/:sessionId`，干净、可分享、浏览器返回键回到对话流。

### 布局

```
┌─────────────────────────────────┐
│ ← 三角函数专项     第3题 / 共5题  │  ← 顶部导航 + 进度
│                                 │
│                                 │
│  题目内容                       │
│  ┌───────────────────────────┐ │
│  │ 已知 sin(x) = 3/5,        │ │
│  │ 且 π/2 < x < π,           │ │
│  │ 求 cos(x) 的值。           │ │
│  └───────────────────────────┘ │
│                                 │
│  作答区                         │
│  ┌───────────────────────────┐ │
│  │                           │ │  ← 输入框 / 选择区
│  │                           │ │
│  └───────────────────────────┘ │
│                                 │
│                                 │
│              [继续]             │  ← 滑到下一题的按钮
│      （或左右滑动切换）          │
└─────────────────────────────────┘
```

### 交互规则

- **逐题推进**：一屏一题，做完滑到下一题。支持左右滑动手势（移动端）和键盘快捷键（桌面端）
- **统一提交**：最后一题出现「提交」按钮，点击统一提交所有答案
- **可回退**：滑动回到之前的题目，之前的作答保留
- **可中断**：退出做题页（返回/关闭）→ 作答自动暂存，下次进入可继续
- **进度指示器**：顶部「第 N 题 / 共 M 题」，M 通常为 5

### 异步预批改（后端）

每道题作答后，后台异步提交 Celery task 批改，结果写入 Redis。提交最后一题时只需等待最后一道的批改结果，其余已就绪。

**前端不感知异步进度**——做题页保持干净，用户只看到"做完 → 提交 → 成绩单"。

## 提交后过渡：迷你成绩单

提交后不直接跳回对话流，先展示成绩单：

```
┌──────────────────────────────────┐
│                                  │
│          🎯  练习完成             │
│                                  │
│       正确率  60%（3/5）          │  ← 大数字，直观
│         用时  4分22秒            │
│                                  │
│  ┌────────────────────────────┐  │
│  │ 三角函数 · 函数性质  ★★☆☆☆  │  │  ← 知识点掌握度
│  │ 三角函数 · 公式变换  ★★★★☆  │  │
│  └────────────────────────────┘  │
│                                  │
│       [查看小宇分析 →]            │  ← 回到对话流
│                                  │
└──────────────────────────────────┘
```

### 成绩单内容

- 正确率（大数字，第一视觉重点）
- 用时
- 知识点分布（每个知识点对应的掌握度星级）
- 「查看小宇分析」按钮 → redirect 到对话流，小宇自动展开分析

## 闭环流程（端到端）

```
学生在对话流
  │
  ▼ 小宇推送练习卡片
「三角函数专项 · 5题 · 预计8分钟」
  │ 点击「开始练习」
  ▼
全屏做题页 /xiaoyu/practice/:id
  │ 逐题推进，后台逐题 Celery 预批改
  │ 5题做完 → 点提交
  ▼
迷你成绩单
  │ 正确率 60% · 用时 4分22秒
  │ 知识点掌握度分布
  │ 点「查看小宇分析」
  ▼
回到对话流 /xiaoyu?practiceResult=:id
  │ 结果卡片出现 + 小宇主动分析错因
  │ 小宇推送纠错卡片
  │ 点「开始纠错」→ 再次全屏做题
  ▼
  ...闭环继续...
```

## 移动端适配

- 对话流：单栏，天然适配手机
- 全屏做题：独立页面，全屏沉浸，左右滑动手势切题
- 成绩单：纵向滚动，适配各种屏幕
- 不依赖拖拽、分屏、悬浮面板

---

## 效率指标（目标）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 闭环时间 | ≤ 3 分钟 | 从点「开始练习」到看到成绩单 |
| 纠错转化率 | ≥ +15% | 纠错后正确率 - 初次正确率 |
| 卡片点击率 | ≥ 60% | 小宇推送练习卡片后被点击的比例 |
| 完成率 | ≥ 80% | 开始做题后全部完成的比例 |

---

## 不做的事

- ❌ 不在对话流内建答题组件（答题在全屏层）
- ❌ 不做 split-view / 侧面板
- ❌ 不做模式切换 UI（小宇自适应）
- ❌ 不做多页面跳转（对话→做题→成绩单→对话，线性闭环）
