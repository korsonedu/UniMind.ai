版本：v2.0（唯一基准）
发布日期：2026-04-30
适用范围：UniMind 全系统（产品、算法、内容、运营、权限与治理）

## 1. 总体架构

UniMind 后续功能统一为六层：
1. 记忆与学习策略层（GFSRS、复习预算、目标保持率）
2. 知识图谱与内容理解层（个性标注、视频语义入图）
3. AI 生产与质控层（单一出题管线、审校、归类）
4. 体验与学习闭环层（学-练-测-复盘-再学）
5. 人员权限与运营治理层（Superuser、RBAC、审计）
6. 稳定性与评估层（模板稳定、质量雷达、实验平台）

## 2. 核心能力与功能清单

## 2.1 记忆与学习策略层

1. `GFSRS-Pro` 自动个性化调参（研究级）：
- 支持用户级参数自适应
- 支持分阶段目标保持率（常规期/冲刺期）
- 支持在线更新与回滚

2. 动态复习预算优化：
- 在固定学习时长约束下优化长期保留
- 按知识点价值、遗忘风险、考试时效联合排序

3. 学习策略教练：
- 每日给出“最小可执行学习建议”
- 提供可解释推荐证据（为什么先学这个）

## 2.2 知识图谱与内容理解层

1. 个性化知识图谱标注：
- 掌握度
- 易错因子
- 迁移能力
- 置信区间

2. 根因知识点定位：
- 自动识别错误背后的先修缺口
- 形成可执行补齐路径

3. 视频语义理解与入图：
- ASR 转写
- 章节切分
- 术语与公式抽取
- 与 431 大纲/知识图谱自动对齐

4. 视频后测与回锚：
- 每段视频结束自动小测
- 错题自动回锚到视频时间片
- 输出二刷路径建议

## 2.3 AI 生产与质控层（单一管线）

仅保留一条题目生产链：
`Author -> Adversarial Reviewer -> Classifier`

1. Author（出题人）
- 新窗口、多模型并行生成题目
- 输出统一 JSON Schema

2. Reviewer（审核人）
- 使用更强模型做对抗评估
- 检查逻辑正确性
- 检查 LaTeX 语法与渲染风险
- 检查是否偏离 431 大纲

3. Classifier（归类人）
- 章节归类
- 难度归类
- 能力维度归类

4. 发布闸门
- 未通过阈值审核的题目禁止入库
- 全链路保留审校证据和回溯记录

## 2.4 体验与学习闭环层

1. 错因闭环引擎：
- 概念错
- 计算错
- 审题错
- 粗心错

2. 自适应组卷引擎：
- 结合用户弱点与考试目标自动出卷
- 支持冲刺模式与稳态模式

3. 复盘自动化：
- 题后自动给出错因、补救路径、下一步任务
- 按周生成进步报告与风险提醒

## 2.5 人员权限与运营治理层

1. Superuser 人员管理：
- 手动分组与标签
- 可配置管理员层级
- 可配置细粒度权限

2. 统一权限模型（RBAC + 标签）
- 角色：学生/教师/教研/审核/运营/管理员
- 权限维度：出题、审题、发布、回滚、导出、模型配置

3. 风险控制与审计：
- 关键操作双人审批
- 全量审计日志
- 高风险变更可回滚

## 2.6 稳定性与评估层

1. Prompt 模板抽取与 DSL 化：
- 角色
- 输入约束
- 输出结构
- 反例约束
- 拒答条件

2. Prompt 版本治理：
- 版本化
- 回归测试
- 灰度与回滚

3. 题目质量雷达：
- 重复率
- 偏纲率
- 区分度
- 误导率
- 解析一致性

4. A/B 与离线回放平台：
- 策略迭代必须先离线评估
- 在线仅小流量灰度

## 3. 新增功能（本轮扩展）

1. 考前智能冲刺模式
- 根据考试日期自动提升保持率目标
- 自动压缩复习区间并控制疲劳

2. 概念漂移监测
- 监测用户长期行为变化
- 触发参数重估与策略迁移

3. 题目生命周期管理
- 生成、审核、上线、复评、退役全流程可追踪

4. 多端一致学习状态
- Web/移动端学习状态一致
- 弱网下支持延迟同步与冲突解决

5. 隐私与合规控制
- 数据脱敏
- 可导出
- 可删除
- 保留策略可配置

## 4. 执行优先级（最新节奏）

## Phase A（0-4 周）
1. AI 单一出题管线上线（Author/Reviewer/Classifier）
2. Prompt 模板版本化与回归测试
3. Superuser 基础权限与人员标签

## Phase B（5-8 周）
1. GFSRS-Pro MVP（离线训练 + 在线推断）
2. 知识图谱个性标注一期
3. 视频语义入图与视频后测闭环

## Phase C（9-12 周）
1. 复习预算优化与动态目标保持率
2. 错因闭环与学习策略教练
3. 题目质量雷达与全链路审计完善

## 5. 验收指标（统一口径）

1. 学习效果
- 7/14/30 天保持率
- 单位学习时长的长期保留增益
- 错因修复率

2. 题库质量
- 审核退回率
- 上线后纠错率
- 偏纲率
- LaTeX 错误率

3. 系统质量
- AI 任务失败率
- 任务平均完成时长
- 高风险操作审计覆盖率

4. 业务指标
- 周有效学习人数
- 学习闭环完成率
- 会员留存与续费率

## 6. 治理规则（执行约束）

1. 新功能必须绑定“可量化指标 + 负责人 + 验收日期”。
2. 算法上线必须经过“离线回放 -> 小流量灰度 -> 全量放开”。
3. 未通过 Reviewer 的题目严禁入库。
4. 权限、题库、模型配置变更必须写入审计日志。
5. 本文件之外的旧计划、旧时间表、旧路线不再生效。
