# UniMind 协作操作规程

## 一、什么放到飞书

- **工作计划、排期、任务状态** → 多维表格「UniMind 工作计划」
- **技术架构、API 约定** → 知识库「UniMind 技术架构」
- **架构决策记录** → 知识库「UniMind 技术架构」下建子页面
- **产品手册、产品介绍、PPT、股权协议、谈判记录、宣传稿** → 知识库「文档归档」下对应子页面
- **产品后续规划、运营后续规划** → 知识库建对应页面
- **日常讨论** → 飞书群（不留需要回溯的信息）

## 二、每个文件放什么

**UniMind 技术架构** — 系统整体架构、技术栈、Agent 设计、AI 模型策略、部署方式、硬边界。架构变更时更新此文档。

**文档归档/** — 按子页面分类存放：
- 产品手册 — 用户使用指南
- 产品介绍 — 对外产品说明
- PPT — 演示文稿
- 股权协议 — 股东相关协议
- 谈判记录 — 商务谈判纪要
- 宣传稿 — 对外宣传素材

**多维表格「UniMind 工作计划」** — 字段说明：
| 字段 | 填写 |
|------|------|
| 任务名称 | 简短，动词开头 |
| 负责人 | 选人 |
| 工作类别 | 按实际分类选择 |
| 优先级 | P0 / P1 / P2 |
| 预计完成时间 | 具体日期 |
| 状态 | 待开始 → 进行中 → 已完成 / 搁置 |
| 关联资源 | 贴链接 |

## 三、协作流程

**每周** — 打开多维表格，过一遍上周任务状态，标记完成、调整未完成。新增本周任务各自认领。状态变了直接改。

**每月** — 30 分钟飞书视频会议。过本月完成情况，对齐下月方向，调整优先级。

**决策** — 涉及方向的选择（非纯执行），在技术架构下建子页面记录：日期、背景、方案对比、选择及原因、影响范围。五句话说清。

**优化建议** — 多维表格新增一条，任务名称写"建议：xxx"，优先级 P2，状态待开始。周回顾时一起过。

**每季度** — 清理知识库，删过期内容，归档旧决策。
