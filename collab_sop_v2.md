# UniMind 协作操作规程

## 一、什么东西放哪里

| 内容类型 | 放哪里 | 具体位置 |
|---------|--------|---------|
| 工作计划、功能排期 | 飞书多维表格 | "UniMind 工作计划"，链接见知识库首页 |
| 技术架构文档 | 飞书知识库 | 技术架构 页面 |
| API 接口约定 | 飞书知识库 | API 约定 页面 |
| 产品后续规划 | 飞书知识库 | 产品后续计划 页面 |
| 运营后续规划 | 飞书知识库 | 运营后续计划 页面 |
| 架构决策记录 | 飞书知识库 | 技术架构 页面内建子页面 |
| 产品手册、PPT、股权协议、谈判记录、宣传稿 | 飞书知识库 | 文档归档 页面，按子页面分类存放 |
| 个人笔记、半成品思考 | Obsidian | 各自本地，不放入知识库 |
| 日常讨论、快速同步 | 飞书群 | 不留需要回溯的信息 |

## 二、多维表格字段说明

表地址：知识库首页找"UniMind 工作计划"链接

| 字段 | 填写方式 |
|------|---------|
| 任务名称 | 简短清晰，动词开头。如"完成知识树导入" |
| 负责人 | 选自己或对方 |
| 优先级 | P0 本周必做 / P1 本月 / P2 后续 |
| 预计完成时间 | 具体日期 |
| 状态 | 待开始 → 进行中 → 已完成 / 搁置 |
| 关联资源 | 贴相关链接（知识库页面、文档、PR等） |

**周流程：** 每周初过一遍，标记已完成的、调整未完成的，新增本周任务。

## 三、怎么写后续计划

**产品后续计划** 和 **运营后续计划** 各自一个页面。按以下结构写：

```
## 近期（本月）
- [ ] 具体事项1 — 负责人 — 预期完成时间
- [ ] 具体事项2

## 中期（下季度）
- [ ] 事项

## 远期 / 待讨论
- 想法或方向，不需要排期
```

每个事项一行，完成就勾掉。新增想法加在对应时间段下。每月回顾时把已完成的清掉，把中期提升为近期。

## 四、怎么提优化建议

在多维表格里新增一条记录：
- 任务名称写"建议：xxx"
- 优先级标 P2
- 状态标"待开始"
- 关联资源贴相关链接
- 不用等对方回应，下次周回顾时一起过

紧急的建议直接群里说。

## 五、决策记录模板

涉及方向的决策（不是纯执行细节），在技术架构页面下建子页面：

```
# 决策：一句话描述

- 日期：
- 背景：
- 方案对比：
- 选择及原因：
- 影响范围：
```

不用写长。目的是半年后翻出来知道当时为什么这么选。

## 六、定期节点

- 每周：多维表格更新
- 每月：30 分钟飞书视频会议，过完成情况 + 对齐下月方向
- 每季度：清理知识库，删过期内容，归档旧决策
