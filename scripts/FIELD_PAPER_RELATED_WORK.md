# Field: Knowledge Graph Diagnostic Engine
## Related Work & Positioning

---

## 1. 问题定义

Field 解决的是一个**新问题**：给定知识图谱 G=(V,E) 和部分观测（学生在少数 KP 上的考试结果），推断全部 |V| 个 KP 的连续掌握度 μ_i ∈ [0,1]。

现有方法没有一个直接解决这个问题。下面逐一比较。

## 3. 为什么没有直接 SOTA 可比

### 3.1 Knowledge Tracing 系列（DKT, DKVMN, AKT, SAKT）

**不同问题。** KT 的输入是学生的完整答题序列 (q1,a1,q2,a2,...)，目标是预测下一题的 P(correct)。Field 的输入是一次性诊断测试的结果，输出是所有 KP 的当前掌握度。KT 方法无法从 20 道题的截面数据推断 200 个 KP——它们需要每 KP 的答题历史来建模状态转移。

### 3.2 Cognitive Diagnosis Models（DINA, NIDA, CDM）

**不同表示。** CDM 输出离散的属性向量 α ∈ {0,1}^K（学生掌握了哪些技能），Field 输出连续掌握度 μ ∈ [0,1]^K。离散化丢失了信息（"掌握了 60%"和"掌握了 90%"在 CDM 中都是 α=1）。且 CDM 需要 Q 矩阵（题目-技能映射），Field 只需要知识图谱。

### 3.3 Graph Neural Networks（GCN, GAT）

**需要训练数据。** GCN 是半监督方法，需要一部分节点的标签（已知 K 值）来训练。Field 是零样本——不依赖任何学生数据，只需要图结构和题目元数据（难度）。在教育场景中，获取"已知 K 值"是不现实的——如果 K 值已知，就不需要诊断了。

### 3.4 Graph Gaussian Process

**太慢。** 图高斯过程可以给出完整后验（均值+方差），但精确推断需要 O(n³)。对于 n=614 KP，这不可行。Field 使用共轭梯度求解，复杂度 O(n√κ)，对于 n=614 只需 <0.1 秒。

### 3.5 Label Propagation

**最强的图基线。** LabelProp 用图平滑（随机游走拉普拉斯）传播标签，但不建模 Bernoulli 观测噪声——它把 0/1 结果当成连续值处理。Field 在此基础上做了两件事：(1) 用 Bernoulli 似然替换高斯似然，(2) 用 GMRF 形式化替换迭代平滑。实验显示 Field 在分层抽样下赢 6-10%。

### 3.6 FSRS

**互补，非竞争。** FSRS 从复习历史估算记忆保留，输入是"何时复习、评级如何"。Field 从考试结果推断掌握度，输入是"测了哪个 KP、对/错"。两者输入不同、输出不同、用途不同：
- FSRS → 找最该复习的 KP（调度）
- Field → 画知识图谱热力图（诊断）
- 组合 → 诊断结果喂给调度器，精准复习

## 4. Field 的独特位置

```
         不需要训练数据          需要训练数据
          ───────────           ───────────
用图      Field ★              GCN / GAT
          LabelProp            Graph GP

不用图    IRT / MIRT            DKT / AKT / SAKT
          Global-Avg           CDM (DINA/NIDA)
```

Field 是唯一同时满足"零样本 + 用图 + 连续 K + 可计算后验方差"的方法。

## 6. 需要在论文中诚实说明的事

1. **调度上 Field 输给 FSRS。** 22 个版本全失败。这不是方法的缺陷——Field 在解决不同的问题。论文应明确区分"诊断"和"调度"。

2. **当观测过于稀疏时（<5% 覆盖 + 非分层），Field 增益有限。** 图不能创造信息——如果某个 topic 没有任何 KP 被测试，Field 只能依赖先验（0.5）。

3. **图结构质量影响性能。** 当前图来自树结构 + LLM 标注的边。更好的图 → 更好的诊断。但即使当前图，Field 已赢 LabelProp。

4. **题目难度 d_i 需要已知。** 生产环境中可从题库元数据获取，或用 IRT 预标定。这是合理的假设。
